# 未来视觉迭代原则

当前仪式台七组视觉目标已接入正式 v1 素材。后续视觉迭代围绕既有图片素材、插槽和分层组件继续演进，不用 CSS 或 SVG 临时重画复杂对象。

素材目录、命名、插槽、manifest 和接入边界见 `docs/ASSET_PIPELINE.md`。视觉组件 props 契约见 `docs/VISUAL_COMPONENT_CONTRACTS.md`。

桌面小窗和 Tauri 外壳迁移见 `docs/TAURI_MIGRATION_PLAN.md`。视觉组件应继续保持平台无关，避免直接依赖 Tauri API。

桌面应用图标、菜单栏图标和通知图标的资产边界见 `src-tauri/icons/README.md`。这些图标属于桌面外壳资产，不应混入符箓、香炉、线香等仪式台视觉组件。

## 视觉状态中间层

业务状态不直接等同于素材状态。当前通过 `src/lib/visualState.ts` 把业务状态映射为视觉状态，再交给 `TalismanVisual`、`CenserVisual`、`IncenseVisual` 使用。

- 香炉视觉状态：`idle` / `active` / `resting` / `completed`
- 线香视觉状态：`pending` / `burning` / `burned` / `resting`
- 符箓视觉状态：`ready` / `disabled` / `completed`

主祭台单槽状态也通过 `getStageIntentVisualSemantics` 收敛，不在组件里散落判断：

- `idle`：情境符箓是启动入口，预防性符箓可见，香炉开盖，线香完整待燃，不显示计时面板。
- `burning`：情境符箓退场，预防性符箓可见，香炉开盖，当前线香按进度燃烧，显示计时面板。
- `resting`：情境符箓保持退场，预防性符箓可见，香炉开盖，当前线香保持上一段已烧完状态，不表现为继续燃烧，显示休息计时面板。
- `completed`：情境符箓保持退场，预防性符箓退场，香炉闭盖并弱化，线香隐藏，不显示计时面板。

主祭台状态标签、香数和线香进度等辅助信息默认隐藏。烧香中或休息中鼠标悬停到香炉命中区时，只保留一个清晰信息框，优先保留带剩余时间的计时卡；不要同时显示内容重复的香炉 metadata 卡片。鼠标悬停到符箓时只放大符箓，不显示香炉辅助信息；鼠标悬停到线香区域时也不显示这些辅助信息。主祭台默认画面应尽量接近真实祭台场景，不把状态文案作为常驻视觉焦点。

主祭台点击入口只保留在情境符箓上。香炉只显示辅助信息，线香只显示进度，预防性符箓只用于查看内容。小窗香炉的点击规则单独处理，只用于展开完整窗口，不和主祭台规则混用。

未来替换真实素材、添加燃烧或开盖动画时，应优先扩展这个视觉状态中间层，而不是让素材组件直接读取完整业务流程。

当前版本已实现分层启动动画和状态落点：第一次确认开始后，情境符箓在约 2 秒内依次表现局部起燃、焦边推进、模板和文字共同灰化、最后收束消失，再正式进入烧香倒计时；`burning` 期间主祭台线香和香炉使用低频呼吸与淡烟漂移；进入 `resting` 时保留短暂余烟和冷却过渡；全部线香烧完后，预防性符箓退场，香炉合盖并弱化，完成卡片克制浮入。同一香炉后续续香不再播放情境符箓燃烧动画。未来可以替换为更真实的符箓燃烧素材细节，但不能改变这条流程契约。

## 符箓

1. 符箓继续使用图片模板作为背景。
2. 不允许用 CSS 或 SVG 随便重新画一个假的符箓。
3. 符箓文字应该是覆盖在图片上的文本层。
4. 情境性符箓和预防性符箓可以使用不同模板，但文本层结构应保持一致，便于组件复用。

情境性符箓和预防性符箓统一从 `src/components/TalismanVisual.tsx` 接入视觉层。后续迭代符箓模板时，优先替换 `TalismanVisual` 内部结构和样式，不改开始确认、计时、休息、续香、复盘等业务逻辑。

符箓 hover/focus 的首要目标是阅读用户填写的执行意图。后续放大可以越过原有舞台层级限制，临时遮蔽其他内容也可接受；但放大态必须仍是同一个 `TalismanVisual`，不能额外复制一套业务入口或把文字渲染到脱离符箓的独立说明卡里。

`TalismanVisual` 当前预留这些图层钩子：

- `data-talisman-layer="template"`：符箓背景模板层
- `data-talisman-layer="text"`：文字覆盖层
- `data-talisman-layer="state"`：状态层，未来可用于燃烧、消失或禁用等视觉状态

## 香炉

香炉当前已经拆成图层：

- 香炉主体
- 香炉盖
- 炉口
- 香灰层
- 底足

香炉开盖动画未来应该移动已有盖子素材，而不是重新画一个盖子。`lid` 必须对应香炉上方完整盖子，包括顶部钮和镂空盖面；`mouth` 只负责盖子下沿和炉口过渡区域。炉口和香灰层需要预留给线香组件定位。

香炉默认开盖：未开始、烧香中、休息中都保持盖子打开，让线香能够插在炉口。只有该香炉对应的全部线香都烧完，也就是执行意图进入 `completed` 后，盖子才闭合。盖子闭合后不再显示线香穿出炉口。当前主祭台闭盖使用已有 `lid` 素材做短促下落过渡；后续如果继续做开盖或合盖动画，也必须遵守这条状态规则。

小窗模式使用克制可爱的 Q 版香炉 v1 素材。小窗 ritual 阶段只显示用户创建的 1-3 个小香炉，并且始终单行并排；不显示主祭台、符箓、历史、设置、标题、意图摘要、说明性状态文案或灰色背景面板。小窗香炉与主祭台香炉一一对应；点击任意小窗香炉只用于展开完整窗口查看或处理完整流程，不直接开始或续香。小香炉 hover/focus 时必须能临时查看剩余时间；烧香中必须有克制烟雾动画。

小窗无原生标题栏时，香炉横排上方仍保留独立透明拖拽热区，只显示一条极弱抓取线，并通过 Tauri 窗口位置会话移动窗口。香炉炉身和线香区域也复用同一套窗口位置会话：普通单击展开完整窗口，从按钮按下后产生超过阈值的移动则拖动小窗并抑制随后一次点击，避免误展开。

主祭台香炉和小窗香炉统一从 `src/components/CenserVisual.tsx` 接入视觉层。后续迭代素材时，优先替换 `CenserVisual` 内部结构和样式，不改计时、开始确认、休息、续香、复盘等业务逻辑。

主祭台香炉使用 `stage` 素材族，小窗 Q 版香炉使用 `compact` 素材族。两套素材可以共享图层命名，但不能混用图片文件；主祭台不复用小窗 Q 版素材，小窗也不显示主祭台素材。

`CenserVisual` 当前预留这些图层钩子：

- `data-censer-layer="lid"`：香炉盖
- `data-censer-layer="mouth"`：炉口
- `data-censer-layer="ash"`：香灰层
- `data-censer-layer="body"`：香炉主体
- `data-censer-layer="feet"`：底足

## 线香

线香当前已经由独立组件分层渲染，包含：

- 香体
- 香灰
- 火星
- 烟雾

线香燃烧进度由倒计时状态驱动。主祭台和小窗已分别接入正式 v1 PNG 图层，继续使用既有香数和倒计时字段提供视觉进度。

线香统一从 `src/components/IncenseVisual.tsx` 接入视觉层。当前主祭台线香使用图片图层和进度数据表达最小燃烧状态：当前线香随进度从顶部转为香灰、可见香体逐步缩短，火星低频呼吸，核心淡烟和两层轻量烟缕错峰漂移；进入休息时火星熄灭并保留短暂余烟；已烧完线香保持灰色；未烧线香保持完整深色。后续小窗 `compact` 也必须显示可辨认但克制的烧香烟雾。主祭台和小窗烟雾都不做香灰掉落、高频闪动或复杂粒子。后续继续优化线香燃烧、香灰、火星、烟雾时，优先替换 `IncenseVisual` 内部结构和样式，不改计时、休息、续香、复盘等业务逻辑。

线香数量必须与用户选择的香数一致。选择 1 / 2 / 3 炷香时，香炉内分别显示 1 / 2 / 3 根线香。进入计时后，从左侧第一根开始显示当前炷香进度；续香后依次推进到第二根、第三根。

`IncenseVisual` 当前预留这些图层钩子：

- `data-incense-layer="stick"`：香体
- `data-incense-layer="ash"`：香灰
- `data-incense-layer="ember"`：火星
- `data-incense-layer="smoke"`：烟雾
- `data-incense-smoke-layer="near"`：主祭台较近的轻量烟缕
- `data-incense-smoke-layer="far"`：主祭台较远、更弱化的错峰烟缕

## 动画边界

动画只用于增强行动仪式感，不应干扰计时、休息、续香、复盘等核心流程。任何动画都应支持关闭或降级，保证桌面小窗长期运行时稳定、安静、低占用。当前主祭台新增动画使用 CSS keyframes 和 transition，并通过 `prefers-reduced-motion` 降级。

情境符箓燃烧动画、预防性符箓退场、香炉合盖和线香燃尽都必须由既有状态和短暂视觉启动状态驱动，不能额外创建一套与业务状态并行的隐藏流程。动画失败或被降级时，界面仍应落到相同结果：情境符箓不可见，已完成套组的预防性符箓不可见，香炉闭盖且弱化。

## 当前 v1 基线

仪式台正式 v1 素材继续遵守既有结构关系：上方情境性符箓、中间香炉与线香、下方预防性符箓。图片不承载动态业务状态，未配置的可选图层仍可使用克制的 CSS 回退。

当前 `src-tauri/icons/app-icon/placeholder-icon.png` 也是占位资产，不代表正式应用图标。macOS 菜单栏右侧已经接入独立的透明单色简化香炉 template icon，不复用应用图标或仪式台分层 PNG。

主祭台第一轮截图审计后的 CSS 基线已经补充中心光、边缘收束和香炉落桌阴影。情境符箓与预防符箓默认降低饱和度，hover 或键盘聚焦时恢复亮度并扩大正文可读区；后续 hover 放大幅度应进一步提高，优先让符箓文字可读，允许临时遮蔽其他内容。`burning` 香炉和线香增加低频暖色呼吸与淡烟漂移，`resting` 增加短暂余烟并降饱和冷却，`completed` 增加合盖、弱化和完成卡片浮入。香炉 hover 后只保留一个信息框；烧香中或休息中优先保留运行态计时卡，避免再显示内容重复的香炉 metadata 卡。
