# 正式视觉素材边界

本文用于对齐最终正式素材的职责边界。它回答一个问题：哪些东西应该画进图片，哪些东西必须继续留在 React 组件、CSS 和业务状态里。

当前项目的七组仪式台素材已完成正式 v1 接入。后续迭代素材前，应先按本文确认边界，再按 `docs/VISUAL_ASSET_REPLACEMENT_CHECKLIST.md` 执行替换。

## 总原则

- 背景只负责环境氛围，不负责业务状态。
- 香炉只负责容器本体，不包含线香、符箓、状态文字或桌面背景。
- 线香只负责线香本体和视觉层，不包含香炉、背景或香数状态。
- 符箓模板只负责固定符纹和纸张质感，不包含用户填写的执行意图文本。
- 计时、完成、休息、复盘、hover 信息、燃烧退场、开盖闭盖语义都由代码控制。

## 主祭台背景

正式背景可以参考当前空桌构图，但不能直接使用带固定香炉的整图作为最终背景。

图片可以包含：

- 墙面、桌面、山水或室内氛围。
- 明确的桌面前沿和桌面承托关系。
- 与整体气质一致的静态装饰。

图片不能包含：

- 固定香炉。
- 符箓、线香、烟雾、火星。
- 已完成、未开始、香数、线香进度等状态文字。
- 按钮、提示框、光效高亮或任何交互提示。

对齐要求：

- 参考线统一维护在 `src/lib/visualAssets.ts` 的 `ALTAR_BACKGROUND_ALIGNMENT_GUIDES`。
- 三个香炉中心应落在同一条水平线上。
- 情境符箓上沿、香炉中心线、预防符箓贴边应能在背景中留出空间。

## 主祭台香炉

主祭台香炉使用 `censer/stage` 素材族，风格可以更正式，但必须分层。

必须提供：

- `lid.png`：完整盖子，包括顶部钮和镂空盖面。
- `mouth.png`：炉口或盖下过渡区域，不承载完整盖子。
- `ash.png`：香灰层。
- `body.png`：香炉主体。
- `feet.png`：底足。

图片不能包含：

- 线香。
- 符箓。
- 桌面背景。
- 状态文字。

状态边界：

- 默认开盖。
- `idle`、`burning`、`resting` 状态都保持开盖。
- 只有该香炉全部线香烧完并进入 `completed` 后才闭盖。
- 闭盖后线香必须隐藏，不能出现线香穿出盖子的画面。
- 完成态可以弱化，但不能改变香炉在桌面上的水平基线。

## 小窗 Q 版香炉

小窗香炉使用 `censer/compact` 素材族，只承担低干扰计时器职责。

图片可以包含：

- 克制可爱的香炉主体。
- 与主祭台对应的盖、炉口、香灰、底足分层。

图片不能包含：

- 白色、灰色、米色或任何窗口背景板。
- 符箓。
- 执行意图文字。
- 状态说明、倒计时文字或香数文字。

交互边界：

- 小窗只显示用户创建的 1-3 个香炉，单行并排。
- 点击香炉只返回完整窗口。
- 小窗不启动套组、不续香、不直接打开复盘。

## 符箓模板

情境性符箓和预防性符箓各使用一套固定模板。模板可以重新设计，但用户文本必须继续由 React 文本层覆盖。

图片可以包含：

- 黄纸质感。
- 固定符纹。
- 边框、印章、装饰纹样。
- 为情境性和预防性区分的固定模板细节。

图片不能包含：

- 用户填写的“当……就……”文本。
- 用户填写的“如果……那么……”文本。
- 燃烧、退场、完成等状态。
- 任何会随用户内容变化的字句。

文字安全区：

- 左右两侧必须留出清晰竖排文字栏。
- 中央符文不得压住左右文字栏。
- 文本安全区维护在 `src/lib/visualAssets.ts` 的 `TALISMAN_TEXT_SAFE_ZONES`。
- 默认尺寸下符箓应弱于香炉；hover 或键盘聚焦时可放大查看。
- 悬停放大后的符箓不能被香炉、线香、桌面边界或其他符箓遮挡。

## 线香

线香使用 `incense/stage` 和 `incense/compact` 两套素材族。

必须提供：

- `stick.png`：香体。
- `ash.png`：香灰。
- `ember.png`：火星。
- `smoke.png`：烟雾。

图片不能包含：

- 香炉。
- 桌面或窗口背景。
- 符箓。
- 状态文字。
- 1 / 2 / 3 炷香的整组烘焙图。

状态边界：

- 线香数量由用户选择的 1 / 2 / 3 炷香决定。
- 从左侧第一炷开始烧，续香后依次推进。
- 当前香灰高度和火星位置由倒计时进度驱动。
- 主祭台 `smoke.png` 是核心烟雾素材；CSS 可以叠加 `near` / `far` 两个轻量烟缕锚点，但不能把它们变成新的素材插槽、业务状态或独立 timer。
- 小窗 `compact` 保持低干扰静态基线，不启用主祭台新增烟缕动画。
- 休息中保持上一炷已烧完状态。
- 香炉闭盖后不显示线香。

## React 必须继续负责的内容

以下内容不能交给静态图片：

- 执行意图文本渲染。
- 当前状态：未开始、烧香中、休息中、已完成。
- 当前第几炷香。
- 线香进度。
- 主祭台线香烟雾的错峰烟缕、休息余烟收束和 `prefers-reduced-motion` 降级。
- 情境符箓首次启动时的燃烧退场。
- 预防符箓完成后的退场。
- 香炉闭盖与完成弱化。
- hover 信息卡和可访问性语义。
- 小窗与完整窗口之间的交互。

## 验收口径

正式素材每替换一组，至少执行：

```bash
npm run check:visual-assets
npm run test -- src/lib/visualAssetManifest.test.ts
npm run build
```

然后打开开发环境“素材”面板，切换香炉状态、香数、当前香、线香进度和预防符箓数量，并打开背景参考线做视觉验收。
