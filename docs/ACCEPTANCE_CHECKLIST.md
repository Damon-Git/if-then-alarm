# 验收清单

每次功能迭代后，至少按本清单手动验收一遍。当前计时使用开发常量：单炷香 10 秒，休息 5 秒。

## 启动与构建

- [ ] 执行 `npm install` 后依赖安装正常。
- [ ] 执行 `npm run dev` 后页面可以打开。
- [ ] 执行 `npm run check:desktop-config` 通过。
- [ ] 执行 `npm run build` 通过。
- [ ] 执行 `npm run test` 通过。
- [ ] 执行 `cargo check` 通过。
- [ ] 执行 `npm run tauri:build` 通过，并生成内部测试 `.app`。
- [ ] 内部测试 `.app` 可以打开，且初始窗口接近 `960px × 760px`。
- [ ] 内部测试 `.app` 保存复盘后重新打开，历史记录仍在。

## 填写页

- [ ] 默认显示 1 套执行意图。
- [ ] 可以添加到最多 3 套。
- [ ] 可以删除多余套组，但至少保留 1 套。
- [ ] 情境性执行意图为空时不能提交。
- [ ] 情境性执行意图不同时包含“当”和“就”时不能提交。
- [ ] 情境性执行意图只有“当”和“就”、但缺少触发情境或行动时不能提交。
- [ ] 情境性执行意图里的多余空格和换行会在提交后被归一化。
- [ ] 情境性执行意图错误显示在对应套组的对应输入框下方。
- [ ] 预防性执行意图为空时可以提交。
- [ ] 预防性执行意图填写后不同时包含“如果”和“那么”时不能提交。
- [ ] 预防性执行意图只有“如果”和“那么”、但缺少阻碍或替代动作时不能提交。
- [ ] 预防性执行意图错误显示在对应套组的对应条目下方。
- [ ] 每套可以选择 1 / 2 / 3 炷香。

## 计时设置

- [ ] 默认使用开发模式：10 秒专注 / 5 秒休息。
- [ ] 设置默认不常驻显示，可通过右上角“设置”打开或收起。
- [ ] 可以切换到正式模式：25 分钟专注 / 5 分钟休息。
- [ ] 切换计时模式后刷新页面，设置仍然保留。
- [ ] 当前轮次进行中时，不能切换计时模式。
- [ ] 有待恢复轮次时，不能切换计时模式。
- [ ] “声音提醒”开关默认关闭。
- [ ] “声音提醒”开关可以保存并在刷新后保持状态。
- [ ] 关闭“声音提醒”时，专注结束、休息结束和整轮完成都不播放声音。
- [ ] 开启“声音提醒”后，专注结束、休息结束和整轮完成各最多播放一次临时钟声。
- [ ] 专注或休息开始时，不播放声音。
- [ ] 历史记录摘要显示本轮使用的计时模式。
- [ ] 开发环境下，设置面板显示“开发恢复场景”。
- [ ] 当前轮次进行中或存在待恢复轮次时，开发恢复场景按钮不可用。
- [ ] Tauri 桌面环境下，设置面板显示“窗口置顶”和“在 Dock 中显示”。
- [ ] 浏览器环境下，设置面板不显示“窗口置顶”和“在 Dock 中显示”。

## 仪式台

- [ ] 提交后进入仪式台。
- [ ] Tauri 桌面环境下，提交后先进入完整窗口和主祭台，不直接进入纯香炉小窗。
- [ ] 仪式台显示对应数量的套组。
- [ ] 桌面宽度下保留完整祭台结构。
- [ ] 完整仪式台使用共享主祭台背景，而不是 1-3 张纵向卡片各自承载全部内容。
- [ ] 主祭台背景不烘焙香炉、符箓或线香，只提供空桌面场景。
- [ ] 创建 1 / 2 / 3 套执行意图时，主祭台香炉分别按 1 / 2 / 3 个槽位在同一张桌子上均匀排列。
- [ ] 主祭台上的 1-3 个香炉始终保持在同一条水平线上，不因完成状态、倒计时或文字显隐上下移动。
- [ ] “已完成”“3 炷香”“线香进度”等说明文字在主祭台默认不显示，只在鼠标悬停或键盘聚焦对应槽位时作为辅助信息显示。
- [ ] 每套上方显示情境性符箓占位。
- [ ] 情境性符箓由 `TalismanVisual` 渲染。
- [ ] 情境性符箓使用 `talisman/situation/template` 图片模板，不退回纯 CSS 占位卡。
- [ ] 情境性执行意图文本以黑色竖排文字覆盖在符箓两侧，不烘焙进图片。
- [ ] 情境性符箓在完整仪式台上保持瘦长比例，视觉宽度明显小于对应香炉。
- [ ] 情境性符箓模板为左右文字栏保留清晰空白，黑色执行意图文本不压住中央符文。
- [ ] 主祭台符箓默认按比例缩小，鼠标悬停或键盘聚焦时可以放大查看。
- [ ] 每套中间显示香炉占位和当前香数。
- [ ] 主祭台香炉由 `CenserVisual` 渲染。
- [ ] 主祭台香炉使用 `stage` 视觉素材族，不复用小窗 `compact` Q 版素材。
- [ ] 主祭台香炉使用 `censer/stage` 临时 PNG 测试素材，不退回 CSS 占位图。
- [ ] 主祭台香炉保留 `body/lid/mouth/ash/feet` 五个图层入口。
- [ ] 主祭台香炉的 `lid` 图层对应完整盖子，包括顶部钮和镂空盖面；不要误放到 `mouth` 或 `body`。
- [ ] 主祭台香炉在未开始、烧香中和休息中保持开盖。
- [ ] 主祭台香炉只有在该套全部线香烧完并进入已完成状态时才闭盖。
- [ ] 主祭台香炉闭盖后，不显示线香穿出已闭合的盖子。
- [ ] 主祭台线香由 `IncenseVisual` 渲染。
- [ ] 主祭台线香使用 `incense/stage` 临时 PNG 测试素材，不退回 CSS 占位图。
- [ ] 主祭台线香保留 `stick/ash/ember/smoke` 四个图层入口。
- [ ] 符箓、香炉、线香视觉状态由 `src/lib/visualState.ts` 统一映射。
- [ ] 符箓、香炉、线香图层带有 `data-visual-slot`，插槽由 `src/lib/visualAssets.ts` 统一生成。
- [ ] 选择 1 / 2 / 3 炷香时，主祭台香炉内分别显示 1 / 2 / 3 根线香。
- [ ] 主祭台线香进度随当前炷香倒计时数据更新。
- [ ] 计时开始后，从左侧第 1 根线香开始显示当前进度；续香后依次推进到第 2、3 根。
- [ ] 每套下方显示预防性符箓占位。
- [ ] 预防性符箓由 `TalismanVisual` 渲染。
- [ ] 预防性符箓使用 `talisman/prevention/template` 图片模板，不退回纯 CSS 占位卡。
- [ ] 预防性执行意图文本以黑色竖排文字覆盖在符箓两侧，不烘焙进图片。
- [ ] 同一套里有 2-3 张预防性符箓时，预防性符箓在香炉下方桌子前沿水平排列，不上下堆叠。
- [ ] 没有预防性执行意图时，主祭台不补空符箓，也不在场景内显示“暂无预防性符箓”。
- [ ] 点击情境性符箓后，先弹出开始确认，不立刻倒计时。
- [ ] 开始确认弹窗显示情境性执行意图、预防性执行意图、香数和计时模式。
- [ ] 点击“取消”后关闭弹窗，对应套组仍保持未开始。
- [ ] 点击“开始这一套”后，对应套组开始倒计时。
- [ ] 有套组倒计时或休息时，其他套组不能开始。

## 小窗香炉舞台

- [ ] 点击“进入仪式台”后先进入完整主祭台，不自动进入小窗香炉舞台。
- [ ] 只有点击“保留并收起”后，才进入小窗香炉舞台。
- [ ] 手动把完整窗口拖到约 390-420px 宽度时，仍停留在完整仪式台，不自动切换为小窗香炉舞台。
- [ ] 显式进入小窗模式后，在 `390px × 620px` 目标尺寸下，页面不出现横向滚动。
- [ ] 小窗 ritual 阶段隐藏应用标题、历史入口、设置入口和仪式台标题。
- [ ] 小窗 ritual 阶段不显示情境性符箓、预防性符箓、计时面板或复盘面板。
- [ ] 小窗 ritual 阶段不显示情境性执行意图摘要。
- [ ] 小窗 ritual 阶段不显示说明性状态文案，只保留香炉场景本身。
- [ ] 小窗 ritual 阶段的 1-3 个香炉始终单行并排。
- [ ] 小窗 ritual 阶段不绘制灰色或米色背景面板。
- [ ] Tauri 小窗 ritual 阶段没有原生标题栏、红黄绿窗口按钮或默认白色窗口底板。
- [ ] 小窗香炉槽位数量与用户创建的套组数量一致，不补空位。
- [ ] 小窗香炉槽位最多显示 3 个。
- [ ] 小窗香炉由 `CenserVisual` 渲染。
- [ ] 小窗线香由 `IncenseVisual` 渲染。
- [ ] 小窗香炉和线香使用 `compact` 视觉素材插槽。
- [ ] 小窗香炉使用 `censer/compact` 临时 PNG 测试素材，不退回 CSS 占位图。
- [ ] 小窗香炉的 `body/lid/mouth/ash/feet` 图层都能正常叠放，没有散开、溢出或明显错位。
- [ ] 小窗香炉内的线香根数与所选香数一致。
- [ ] 小窗未开始香炉视觉弱于正在进行的香炉，但仍可识别。
- [ ] 小窗正在进行的香炉在并排香炉中更明显，但不显示文字、不新增背景面板。
- [ ] 小窗休息中和已完成香炉有弱化状态，线香不应看起来仍在积极燃烧。
- [ ] 小窗香炉在未开始、烧香中和休息中保持开盖；已完成后闭盖并隐藏线香。
- [ ] 未开始的小窗香炉可点击，但只尝试展开完整窗口，不进入开始确认弹窗。
- [ ] 未开始的小窗香炉被点击后，不改变任何套组状态。
- [ ] 烧香中或休息中的小窗香炉点击后尝试展开完整窗口，不能启动其他套组。
- [ ] 已完成的小窗香炉点击后尝试展开完整窗口，不能再次开始。
- [ ] 小窗中最后一炷香结束后，只显示已完成香炉，不直接弹出复盘页。
- [ ] 小窗中全部完成后，点击香炉展开完整窗口，再进入复盘页。
- [ ] 小窗中全部完成后，如果完整窗口展开失败，仍停留在小窗完成态，不进入复盘页，并可再次点击重试。
- [ ] 小窗中全部完成后，快速重复点击香炉不会造成重复状态跳转或重复错误提示。
- [ ] 小窗中全部完成后，香炉按钮语义提示“点击展开完整窗口并复盘”。
- [ ] 小窗下未开始且可点击的香炉通过按钮语义保留“点击展开完整窗口”。
- [ ] 小窗下烧香中的香炉通过按钮语义保留“第 n / m 炷，剩余 mm:ss”。
- [ ] 小窗下休息中的香炉通过按钮语义保留“休息剩余 mm:ss”。
- [ ] 小窗 ritual 阶段不显示大标题、英文 eyebrow 或放弃本轮按钮。
- [ ] 小窗下休息/续香弹窗按钮纵向排列且不拥挤。
- [ ] 小窗非 ritual 阶段的历史和设置面板可以纵向滚动，按钮和筛选控件不横向溢出。
- [ ] 小窗非 ritual 阶段的长执行意图、历史摘要和复盘文本会换行或截断，不遮挡其他控件。
- [ ] Tauri 桌面初始窗口尺寸接近 `960px × 760px`。
- [ ] Tauri 桌面完整窗口尺寸接近 `960px × 760px`。
- [ ] Tauri 桌面小窗模式尺寸接近 `390px × 620px`。
- [ ] “保留并收起”会从完整窗口切回小窗模式，并继续计时。
- [ ] macOS 菜单栏右侧“令”入口可以隐藏并重新打开窗口，不改变当前尺寸模式。

## 烧香与休息

- [ ] 单炷香倒计时结束后弹出“这一炷香已经烧完。”。
- [ ] 还有下一炷香时，可以选择“立刻续香”。
- [ ] 还有下一炷香时，可以选择“休息 5 分钟”。
- [ ] 选择休息后进入休息倒计时。
- [ ] 休息结束后弹出“休息结束，是否继续下一炷香？”。
- [ ] 点击“开始下一炷香”后进入下一炷香倒计时。
- [ ] 最后一炷香结束后，该套显示“已完成”。
- [ ] 所有套组完成后进入复盘页。
- [ ] 计时进行中刷新并恢复时，剩余时间按真实经过时间重新计算。
- [ ] 专注计时已在离开期间结束且还有下一炷香时，恢复后直接显示烧完提醒。
- [ ] 休息计时已在离开期间结束时，恢复后直接显示休息结束提醒。
- [ ] 放弃本轮确认弹窗打开时，当前倒计时冻结；取消放弃后继续计时。

## 恢复场景 Fixture

- [ ] 写入“专注中，剩余 8 秒”后刷新，恢复后继续烧香倒计时。
- [ ] 写入“专注已超时，还有下一炷香”后刷新，恢复后直接显示“这一炷香已经烧完。”。
- [ ] 写入“休息中，剩余 4 秒”后刷新，恢复后继续休息倒计时。
- [ ] 写入“休息已超时”后刷新，恢复后直接显示“休息结束，是否继续下一炷香？”。
- [ ] 写入“最后一炷已超时”后刷新，恢复后进入复盘页。
- [ ] 写入“旧版 session，无计时段”后刷新，恢复后按保存的剩余秒数继续。

- [ ] 复盘页可以选择“部分完成”。
- [ ] 复盘页可以选择“未完成”。
- [ ] 复盘页可以填写主要阻碍。
- [ ] 复盘页可以填写下次调整。
- [ ] 复盘页显示本轮计时模式、专注时长和休息时长。
- [ ] 复盘页可以展开查看每套情境性执行意图、预防性执行意图和香数。
- [ ] 一句复盘为空时不能保存。
- [ ] 保存复盘后返回填写页。
- [ ] 历史记录能看到本次执行意图、香数、完成度和复盘文本。
- [ ] 历史记录默认不常驻显示，可通过右上角“历史”打开或收起。
- [ ] 刷新页面后，历史记录仍然存在。
- [ ] 历史记录默认显示摘要，不直接展开所有执行意图。
- [ ] 历史摘要显示套组数、总香数、约专注时长、完成度和复盘。
- [ ] 历史摘要在有填写时显示主要阻碍和下次调整。
- [ ] 历史分析显示总轮次、完成 / 部分完成 / 未完成次数。
- [ ] 历史分析显示总香数、累计专注分钟和平均每轮香数。
- [ ] 历史分析显示主要阻碍 Top 5。
- [ ] 历史分析显示最近 3 条下次调整。
- [ ] 可以搜索历史记录里的执行意图、阻碍和复盘文本。
- [ ] 可以按完成 / 部分完成 / 未完成筛选历史记录。
- [ ] 可以按全部时间 / 近 7 天 / 近 30 天筛选历史记录。
- [ ] 历史分析会跟随当前筛选结果更新。
- [ ] 没有匹配筛选结果时，显示空结果提示。
- [ ] 展开历史记录后，可以看到每套情境性执行意图和预防性执行意图。
- [ ] 可以导出历史 JSON 文件。

- [ ] 可以导入本应用导出的历史 JSON 文件。
- [ ] 导入重复历史时，会按 id 跳过重复记录。
- [ ] 导入成功后，会用 toast 提示新增和跳过的记录数量。

- [ ] 可以清空全部历史记录。
- [ ] 历史记录不提供“再次使用”或一键复用入口。

## 可用性保护

- [ ] 填写页已有草稿内容时，刷新或关闭页面会出现浏览器离开提示。
- [ ] 填写页只有默认空白套组时，刷新或关闭页面不会出现离开提示。
- [ ] 仪式台点击“放弃本轮”会弹出确认。
- [ ] 取消放弃后仍停留在当前轮次。
- [ ] 确认放弃后回到填写页，且不写入历史。
- [ ] 复盘页点击“放弃本轮”会弹出确认。
- [ ] 仪式台刷新或关闭页面时出现浏览器离开提示。
- [ ] 复盘页刷新或关闭页面时出现浏览器离开提示。
- [ ] 仪式台刷新后，再打开页面会提示恢复或丢弃本轮。
- [ ] 恢复弹窗显示当前阶段、总香数、完成套组数和剩余时间。
- [ ] 点击“恢复本轮”后回到刷新前的仪式台状态。
- [ ] 点击“丢弃本轮”后回到填写页，且不会写入历史。
- [ ] 复盘页刷新后，再打开页面会提示恢复或丢弃本轮。
- [ ] 保存复盘后，再刷新页面不会提示恢复旧轮次。
- [ ] 确认放弃本轮后，再刷新页面不会提示恢复旧轮次。
- [ ] Tauri 桌面环境下，点击窗口关闭按钮时使用应用内确认弹窗，而不是直接关闭。
- [ ] Tauri 桌面环境下，进行中轮次可以选择“放弃并退出”，关闭程序且不恢复本轮。
- [ ] Tauri 桌面环境下，进行中轮次可以选择“保留并收起”，并切回小窗模式。

## 桌面通知与窗口行为

- [ ] 一炷香倒计时结束时，系统通知显示“这一炷香已经烧完。”。
- [ ] 小窗中最后一炷香结束且本轮全部完成时，系统通知提示“本轮已完成，点击香炉回到完整窗口复盘。”。
- [ ] 休息倒计时结束时，系统通知显示“休息结束，是否继续下一炷香？”。
- [ ] 专注或休息开始时，不应立刻弹出结束通知。
- [ ] 开启“窗口置顶”后，窗口保持在其他窗口上方。
- [ ] 关闭“窗口置顶”后，窗口恢复普通层级。
- [ ] 关闭“在 Dock 中显示”后，Dock 图标消失，但菜单栏“令”入口仍可重新打开窗口。
- [ ] 重新开启“在 Dock 中显示”后，Dock 图标恢复。
