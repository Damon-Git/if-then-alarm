# Tauri 关闭与恢复说明

本文记录桌面小窗阶段对“关闭窗口”和“恢复本轮”的产品语义、当前实现和后续迁移方向。目标是避免用户开始一轮仪式后，因为关闭、隐藏、重启或睡眠造成状态含义不清。

## 产品语义

当前阶段采用以下语义：

- 进入仪式台或复盘页后，本轮被视为未保存轮次。
- 关闭窗口不等于完成，也不等于正式放弃。
- 只要本轮未保存复盘，就应该保留恢复或丢弃的选择。
- 计时以真实时间戳推导，不依赖窗口一直打开。
- 用户明确点击“放弃本轮”或“丢弃本轮”时，才清除当前轮次。
- 保存复盘后，本轮写入历史，并清除当前轮次。

这和产品目标一致：仪式感不是强迫用户保持窗口打开，而是让用户清楚知道“我开始了什么、现在是否还在继续、是否要放弃”。

## 当前实现

当前 Web/Tauri MVP 使用同一套核心 session 逻辑，并在桌面环境补充原生关闭拦截：

- `src/App.tsx` 在 `ritual` 和 `review` 阶段保存当前 session。
- `src/lib/sessionStorage.ts` 通过 `jiji-rululing.current-session` 保存未完成轮次。
- `src/lib/timer.ts` 使用 `startedAt` 和 `durationSeconds` 推导剩余时间。
- `RestoreSessionModal` 在启动时提示用户恢复或丢弃未完成轮次。
- 保存复盘、确认放弃本轮、丢弃恢复提示时，会清除当前 session。
- `src/lib/tauriWindow.ts` 封装 Tauri 窗口关闭事件和窗口销毁调用。
- `src/lib/notificationAdapter.ts` 在每段专注/休息开始时设置延迟通知，倒计时结束后再发送系统通知。
- `src-tauri/src/main.rs` 创建菜单栏入口，用户可以通过它显示或隐藏主窗口。
- Tauri 桌面端点击关闭按钮时，如果存在填写草稿、仪式台轮次或复盘轮次，会先展示应用内确认弹窗。

浏览器离开保护仍使用 `beforeunload`：

- `src/App.tsx`：仪式台和复盘页。
- `src/components/SetupForm.tsx`：填写页已有草稿时。

`beforeunload` 只作为 Web 版保护层保留。Tauri 桌面端以窗口关闭事件为准。

桌面关闭确认规则：

- 填写页已有草稿时，提示用户继续填写或关闭窗口；关闭后草稿不会保存。
- 仪式台或复盘页存在未保存轮次时，提示用户继续当前轮次或保留并收起。
- “保留并收起”不会放弃轮次；应用进程继续运行，当前段计时结束仍应收到提醒。
- 用户可以通过菜单栏入口重新打开已收起的窗口。
- 如果用户真正退出应用，再重新启动后仍然走恢复/丢弃流程。

## 恢复规则

启动时如果存在 `jiji-rululing.current-session`：

- 如果正在烧香且剩余时间仍大于 0，恢复后继续倒计时。
- 如果正在烧香且已经超时，恢复后显示“这一炷香已经烧完。”。
- 如果正在休息且剩余时间仍大于 0，恢复后继续休息倒计时。
- 如果正在休息且已经超时，恢复后显示“休息结束，是否继续下一炷香？”。
- 如果最后一炷香已经超时，恢复后进入复盘页。
- 如果已经在复盘页，恢复后回到复盘状态，等待用户保存。

计时不会因为窗口关闭而暂停。后续如果要支持“关闭即暂停”，必须明确改变产品语义并更新状态机。

## 需要手动验收的场景

1. 在烧香倒计时中关闭 Tauri 窗口，再重新启动。
2. 在烧香倒计时快结束前关闭，等待超过剩余时间后重新启动。
3. 在休息倒计时中关闭，重新启动。
4. 在休息倒计时快结束前关闭，等待超过剩余时间后重新启动。
5. 在最后一炷香快结束前关闭，等待超过剩余时间后重新启动。
6. 在烧香或休息倒计时中点击“保留并收起”，等待计时结束，确认系统通知出现。
7. 在复盘页关闭，重新启动。
8. 点击恢复后，确认状态和倒计时合理。
9. 点击丢弃后，确认回到填写页且不写入历史。
10. 保存复盘后，重新启动不再提示恢复旧轮次。

完整桌面验收清单见 `docs/TAURI_DESKTOP_CHECKLIST.md`。

## 后续 Tauri 原生方案

当未来接入菜单栏小窗后，还需要区分：

- 关闭窗口。
- 隐藏窗口。
- 点击菜单栏图标收起。
- 应用真正退出。

这些行为不能混用同一个含义。建议默认把“隐藏/收起”理解为继续计时，把“放弃本轮”保留为显式操作。

## 当前不做

- 不处理 macOS 睡眠后的系统通知补偿。
- 不区分窗口隐藏和应用退出。
- 不改变现有 localStorage session schema。
- 不做通知按钮交互。
