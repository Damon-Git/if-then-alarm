# 声音提醒规格

本文记录钟声提醒的产品边界。当前版本使用 Web Audio 解码并播放随应用打包的本地 WAV，不依赖网络音频。

## 当前状态

- 设置面板提供“声音提醒”开关。
- 默认关闭。
- 开关会保存到应用设置 `isSoundReminderEnabled`。
- 开启后，专注结束、休息结束和整轮完成会播放对应的克制钟声。
- 关闭后不播放声音。
- 设置持久化结构不变，旧数据缺少 `isSoundReminderEnabled` 时仍回退为关闭。

## 播放规则

- 每段专注结束最多响一次。
- 每段休息结束最多响一次。
- 整轮完成最多响一次。
- 不在专注或休息开始时响。
- 不与同一事件的其他声音重复播放。
- 用户关闭“声音提醒”后，所有钟声都不播放。
- 播放失败时静默跳过，不影响计时、弹窗和系统通知。
- 新定时段开始、放弃本轮、保存复盘或丢弃恢复轮次时，取消旧定时器并停止仍在播放的旧钟声。
- 打开“放弃本轮”确认框暂停计时时，提醒定时器也暂停；取消放弃后按剩余时间重新调度。
- 关闭声音开关时立即停止仍在播放的钟声；不改动当前计时段。

## 声音原则

- 声音以克制钟声为主，不做复杂音效。
- 音量应偏低，避免打断用户。
- macOS 勿扰、系统静音和通知设置的行为需要单独验收。

## 正式素材

当前三段素材位于 `src/assets/sounds/`：

| 文件 | 语义 | 节奏 | 时长 |
| --- | --- | --- | --- |
| `incense-finished.wav` | 单炷线香结束 | 单次短击 | `1.30s` |
| `rest-finished.wav` | 休息结束 | 两次短击 | `1.65s` |
| `ritual-completed.wav` | 整轮完成 | 三次克制收束 | `2.45s` |

素材由 `scripts/generate-sound-reminder-assets.mjs` 在本地确定性生成：单声道、`48kHz`、`16-bit PCM WAV`，使用偏小型铜钟的加法共振和短促软化击槌瞬态。生成过程不包含第三方录音、采样或下载素材，因此可以随本应用一同分发。未来如果独立公开发布音频文件，再单独定义公共资产许可。

运行以下命令可重新生成和检查素材：

```bash
npm run generate:sound-assets
npm run check:sound-assets
```

声音播放逻辑继续独立于系统通知；系统通知负责提醒可见性，声音提醒只作为可关闭的辅助反馈。自动检查可以覆盖触发次数、取消逻辑、文件格式、时长和构建引用，但听感仍需要长期自用试听确认。
