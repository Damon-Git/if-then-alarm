# Tauri 桌面验收清单

本清单用于每次修改 Tauri 外壳、小窗布局、弹窗、滚动、通知或桌面相关行为后做手动验收。当前阶段只验收内部 `.app` 冒烟，不验收 DMG、签名、公证或正式发布。

桌面行为的组合回归顺序见 `docs/DESKTOP_BEHAVIOR_REGRESSION.md`。

## 验收前准备

- [ ] 当前工作区没有未保存的重要改动。
- [ ] 执行 `npm run check:desktop-config` 通过。
- [ ] 执行 `npm run test` 通过。
- [ ] 执行 `npm run build` 通过。
- [ ] 执行 `cargo check` 通过。
- [ ] 如果 `5173` 被其他应用占用，先释放端口；如果是本项目 Vite 服务，可直接复用。

## 内部打包冒烟

内部打包说明见 `docs/MACOS_INTERNAL_BUILD.md`。

- [ ] 执行 `npm run tauri:build` 通过。
- [ ] 生成 `src-tauri/target/release/bundle/macos/急急如律令.app`。
- [ ] 当前只验收内部 `.app`，不验收 DMG、签名、公证或正式发布。

## 启动桌面窗口

```bash
npm run tauri:dev
```

- [ ] 命令能启动 Tauri 桌面窗口。
- [ ] 窗口标题显示“急急如律令”。
- [ ] 初始窗口接近 `390px × 620px`。
- [ ] 页面内容加载的是本项目，而不是其他占用 `5173` 的页面。
- [ ] 首次编译后，再次启动速度明显变快。

## 小窗基础布局

- [ ] 小窗下没有横向滚动条。
- [ ] 应用内标题区不挤占过多第一屏高度。
- [ ] 填写页、复盘页和完整状态下，“历史”和“设置”按钮可见且可点击。
- [ ] 小窗 ritual 阶段隐藏“历史”和“设置”按钮。
- [ ] 当前计时模式显示清楚，但不喧宾夺主。
- [ ] 页面主体可以纵向滚动，滚动时没有内容被系统标题栏遮住。

## 窗口行为

- [ ] Tauri 桌面环境下，设置面板显示“窗口置顶”开关。
- [ ] Tauri 桌面环境下，设置面板显示“在 Dock 中显示”开关。
- [ ] 浏览器环境下，设置面板不显示“窗口置顶”和“在 Dock 中显示”开关。
- [ ] 开启“窗口置顶”后，当前窗口保持在其他窗口上方。
- [ ] 关闭“窗口置顶”后，当前窗口恢复普通层级。
- [ ] 重启后，“窗口置顶”开关保留上次设置。
- [ ] 默认情况下，“在 Dock 中显示”处于开启状态。
- [ ] 关闭“在 Dock 中显示”后，Dock 中不再显示应用图标。
- [ ] Dock 图标隐藏后，仍可通过菜单栏“令”入口重新打开窗口。
- [ ] 重新开启“在 Dock 中显示”后，Dock 中恢复显示应用图标。
- [ ] 重启后，“在 Dock 中显示”开关保留上次设置。

## 菜单栏入口

- [ ] macOS 菜单栏右侧状态区能看到“令”字入口。
- [ ] 当前“令”字入口被视为临时菜单栏入口，不验收为正式图标。
- [ ] 点击右侧“令”字入口，当前窗口会隐藏。
- [ ] 再次点击右侧“令”字入口，当前窗口会重新显示并获得焦点。
- [ ] 计时中点击“保留并收起”后，可以通过菜单栏入口重新打开窗口。
- [ ] 通过菜单栏入口隐藏窗口后，当前计时和系统通知不受影响。

## 图标资产

- [ ] `src-tauri/icons/app-icon/placeholder-icon.png` 仍是临时占位应用图标。
- [ ] 应用图标、菜单栏图标和通知图标的职责已在 `src-tauri/icons/README.md` 和各子目录 README 中分开记录。
- [ ] 当前阶段不把“令”字入口当作正式菜单栏 template icon。

## 填写页

- [ ] 默认显示 1 套执行意图。
- [ ] 可以添加到 3 套执行意图。
- [ ] 长情境性执行意图输入时，输入框和页面滚动正常。
- [ ] 校验错误显示后不会撑出横向滚动。
- [ ] 提交后能进入仪式台。

## 小窗仪式台

- [ ] 创建 1 套时显示 1 个香炉。
- [ ] 创建 2 套时显示 2 个香炉。
- [ ] 创建 3 套时显示 3 个香炉。
- [ ] 小窗下显示香炉舞台，不显示完整主祭台三栏布局。
- [ ] 香炉内线香根数与所选香数一致。
- [ ] 线香不会顶出香炉槽位或背景框。
- [ ] 点击未开始香炉可以打开开始确认弹窗。
- [ ] 小窗下烧香中、休息中或已完成的香炉点击后展开并聚焦完整窗口。
- [ ] 小窗下已有活动套组时，点击其他未开始香炉不会启动新套组，只展开完整窗口。
- [ ] 小窗 ritual 阶段不显示状态文案和长执行意图摘要。
- [ ] 小窗 ritual 阶段的香炉按钮仍通过 aria-label 保留当前状态、香数和倒计时。

## 弹窗

- [ ] 开始确认弹窗不会超出窗口。
- [ ] 开始确认弹窗内容过长时可以滚动到底部按钮。
- [ ] 单炷结束弹窗按钮显示完整。
- [ ] 休息结束弹窗按钮显示完整。
- [ ] 放弃本轮确认弹窗显示完整。
- [ ] 历史删除或清空确认弹窗显示完整。
- [ ] 弹窗打开时，背景内容不会横向溢出。

## 计时流程

- [ ] 点击开始后，倒计时正常刷新。
- [ ] 单炷结束后弹出休息或续香选择。
- [ ] 选择休息后进入休息倒计时。
- [ ] 休息结束后可以继续下一炷香。
- [ ] 多炷香会按第 1、2、3 炷依次推进。
- [ ] 所有香完成后进入复盘页。

## 系统通知

- [ ] 首次开始计时时，macOS 会请求通知权限，允许后继续当前流程。
- [ ] 一炷香计时结束时，系统通知显示“这一炷香已经烧完。”。
- [ ] 休息计时结束时，系统通知显示“休息结束，是否继续下一炷香？”。
- [ ] 开始下一段计时后，不会继续保留上一段的待触发通知。
- [ ] 放弃本轮、保存复盘或丢弃恢复轮次后，不会继续触发旧通知。
- [ ] 进行中轮次点击“保留并收起”后，如果当前段计时结束，系统仍能发出通知。

## 复盘与历史

- [ ] 复盘页在小窗下可以滚动到保存按钮。
- [ ] 复盘长文本不会撑出横向滚动。
- [ ] 保存后历史记录出现本轮记录。
- [ ] 重启后，历史记录仍能查看。
- [ ] 历史面板可以滚动查看多条记录。
- [ ] 历史筛选、搜索、导入、导出按钮在小窗非 ritual 阶段不拥挤到不可用。
- [ ] 点击“导出历史”会打开 macOS 保存面板。
- [ ] 取消保存面板后，不显示导出成功 toast。
- [ ] 选择保存路径后，会写入本应用历史 JSON。
- [ ] 点击“导入历史”会打开 macOS 打开文件面板。
- [ ] 取消打开文件面板后，不显示导入失败 toast。
- [ ] 选择本应用导出的 JSON 后，能完成导入并显示新增/跳过数量。
- [ ] 设置面板在小窗非 ritual 阶段可以完整滚动查看。

## 关闭与恢复

关闭与恢复的产品语义见 `docs/TAURI_CLOSE_RESTORE_NOTES.md`。

- [ ] 填写页已有草稿时点击窗口关闭按钮，会出现应用内确认弹窗。
- [ ] 草稿关闭确认中点击“继续填写”，窗口保持打开。
- [ ] 草稿关闭确认中点击“关闭窗口”，窗口关闭，重新启动后不恢复草稿。
- [ ] 仪式台进行中点击窗口关闭按钮，会出现应用内确认弹窗。
- [ ] 进行中关闭确认里点击弹窗关闭按钮，窗口保持打开，当前轮次不变化。
- [ ] 进行中关闭确认里点击“放弃并退出”，窗口关闭，重新启动后不恢复本轮且不写入历史。
- [ ] 进行中关闭确认里点击“保留并收起”，窗口收起且计时继续。
- [ ] 重新启动后，如果有未完成轮次，会出现恢复提示。
- [ ] 点击恢复后能回到当前轮次。
- [ ] 点击丢弃后回到填写页，不写入历史。
- [ ] 保存复盘后重新启动，不会提示恢复旧轮次。

## 持久化迁移准备

- [ ] 浏览器环境下，历史、session、设置仍走 Web `localStorage` adapter。
- [ ] Tauri 环境下，历史、session、设置会写入 app data 目录下的 `persistence.v1.json`。
- [ ] `createMemoryPersistenceAdapter` 作为 Tauri 桌面存储的同步缓存层，业务模块不直接调用异步文件 API。
- [ ] `PERSISTENCE_STORAGE_KEYS` 包含 history、current session、settings 三个 key。
- [ ] 桌面 JSON 纯函数 schema 已覆盖默认值、迁移、坏数据降级和 snapshot 映射。
- [ ] 首次运行 Tauri 桌面版时，如果没有桌面 JSON，会从旧 `localStorage` 生成第一份文件。
- [ ] 从旧 `localStorage` 迁移成功时，会显示“已迁移到桌面本地数据文件。”。
- [ ] 桌面 JSON 损坏时，会先备份为 `persistence.v1.corrupt-*.json`，再尝试恢复。
- [ ] 桌面 JSON 损坏并恢复时，会显示“本地数据文件异常，已保留备份并尝试恢复。”。
- [ ] 桌面持久化初始化失败时，会显示“桌面数据文件暂不可用，本次将临时使用浏览器存储。”。
- [ ] 桌面 JSON 写入失败时，会显示“本地数据文件写入失败，本次更改可能暂未落盘。”。
- [ ] 持久化迁移计划记录在 `docs/PERSISTENCE_MIGRATION_PLAN.md`。
- [ ] 桌面 JSON 文件规格记录在 `docs/DESKTOP_PERSISTENCE_JSON_SPEC.md`。
- [ ] 第一版桌面落盘目标是 app data 目录下的 `persistence.v1.json`。
- [ ] macOS 开发环境可在 `~/Library/Application Support/com.damon.jijirululing/persistence.v1.json` 查看桌面 JSON。
- [ ] 当前阶段不清理旧 `localStorage` 数据。
