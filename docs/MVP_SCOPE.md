# MVP 范围

《急急如律令》的第一轮 MVP 目标是跑通从任务创建到本地历史保存的完整闭环。当前版本同时保留 Web App 开发入口和 Tauri 桌面入口；Tauri 入口已经具备 macOS 小窗、菜单栏入口、系统通知、桌面 JSON 持久化、系统文件对话框、窗口置顶和 Dock 显示设置。

Tauri 迁移路径见 `docs/TAURI_MIGRATION_PLAN.md`。当前阶段可以生成内部测试 `.app`，但仍不做 DMG、签名、公证、发布渠道和完整发布 iconset。

小窗目标尺寸和验收约束见 `docs/COMPACT_WINDOW_SPEC.md`。Tauri 桌面窗口启动时使用 `960px × 760px` 完整窗口基线；显式“保留并收起”后才进入 `390px × 620px` 小窗基线。Web 窄窗口只用于普通响应式排版验证，不自动代表小窗模式。

## 当前要做

- 用户可以创建 1-3 项任务。
- 每项任务包含 1 条目标性执行意图、0-3 条预防性执行意图、1-3 炷香。
- 目标性执行意图使用“当……就……”句式，并做轻校验、空白归一化和字段级错误定位。
- 预防性执行意图使用“如果……那么……”句式；未填写时跳过校验，填写后做空白归一化和字段级错误定位。
- 点击“开始创造”后打开仪式台，显示目标性符箓、香炉、预防性符箓和线香视觉层。
- 点击目标性符箓后先确认本项任务内容，再开始当前炷香倒计时。
- 单炷香结束后弹出休息或续香选择。
- 休息结束后提醒继续下一炷香。
- 全部香完成后进入复盘流程，可回看本轮详情，并记录完成度、主要阻碍、下次调整和一句复盘。
- Web 环境下复盘保存到 localStorage；Tauri 环境下复盘保存到 app data 目录下的桌面 JSON。
- 刷新页面或重启桌面应用后仍能查看历史记录。
- 历史记录以折叠摘要展示复盘字段，支持删除单条和清空全部。
- 历史记录顶部展示轻量分析：总轮次、完成度、总香数、累计专注、阻碍 Top 5 和最近下次调整。
- 历史记录支持搜索、完成度筛选和时间范围筛选，分析面板会跟随筛选结果更新。
- 历史记录支持导出 JSON 备份，并支持导入本应用导出的 JSON 文件。
- 复盘保存、历史导入导出、删除和清空使用轻量 toast 提示。
- 支持开发模式和正式模式切换，计时模式保存到 localStorage，并写入当前 session 和历史记录。
- 设置中提供声音提醒开关，默认关闭；开启后使用 Web Audio 解码并播放随应用打包的克制钟声。
- 历史和设置收为右上角弱入口，默认不挤占主流程。
- 点击“开始创造”后进入完整窗口和主祭台；手动拖窄完整窗口不自动切换为小窗。
- 显式选择“保留并收起”后才切换为小窗香炉舞台，只显示用户创建的 1-3 个香炉槽位。
- 小窗香炉槽位只作为低干扰计时器和返回完整窗口的入口；点击后尝试展开完整窗口，不直接开始任务。
- 小窗下隐藏大标题、英文标识和次要操作，只保留低干扰香炉计时场景；鼠标悬停到小香炉时可以显示当前剩余时间，平时不常驻显示。
- 小窗被用户拖动后，点击香炉展开完整窗口再重新“保留并收起”时，应回到上一次小窗位置，不重新居中丢失用户摆放。
- 小窗烧香中必须有克制烟雾动画，作为低干扰计时反馈；休息中和完成态仍需弱化或收束，不残留高负载烟雾。
- 主祭台和小窗香炉统一通过 `CenserVisual` 渲染，并分别使用 `stage` 和 `compact` 正式 v1 素材族。
- 目标性和预防性符箓统一通过 `TalismanVisual` 渲染，并使用两套正式 v1 图片模板。
- 线香统一通过 `IncenseVisual` 渲染，并接收由倒计时计算出的当前炷香进度。
- 视觉组件通过 `src/lib/visualState.ts` 接收语义化视觉状态，业务状态不直接绑定素材状态。
- 视觉素材目录和插槽通过 `src/lib/visualAssets.ts` 约定；主祭台背景、主祭台香炉、小窗 Q 版香炉、两套符箓模板、主祭台线香和小窗线香七组目标均已接入正式 v1。
- 主祭台已完成第一轮截图审计和 CSS 优化：背景增加中心光和边缘收束，符箓默认降饱和并强化 hover 阅读区，香炉与线香的 `burning` / `resting` / `completed` 层级更清楚，香炉信息卡和运行态计时卡避开关键视觉。
- 主祭台已完成第一轮仪式动画优化：`burning` 增加低频火星、分层淡烟和香炉暖色呼吸，`resting` 增加短暂余烟和冷却过渡，`completed` 增加合盖、弱化和完成卡片落幕；新增动画支持 `prefers-reduced-motion` 降级。
- Demo 前符箓燃烧、线香烟雾增强和固定展示路径视觉回归已完成：情境符箓首次启动使用局部起燃、焦边推进、模板和文字共同灰化、最后收束消失的分层退场；主祭台 `stage` 烟雾使用核心 PNG 和两层错峰轻量烟缕，在正常观看距离下保持可辨认但克制的缓慢漂移；单香炉 hover 下计时卡和香炉信息卡已错开。Web 与 Tauri 开发版均已走通主流程，小窗烧香中收起、拖拽和点击香炉展开已复检。完整窗口恢复时会按原生装饰尺寸补偿 client area，连续两次原生往返均稳定恢复为 `960px × 760px` 外框，本次修改结果已由用户验证通过。
- 下一轮符箓燃烧精修需要增加可辨认的小火苗：火苗沿局部起燃点和焦边推进方向短促出现，在约 2 秒退场内完成点燃、摆动和收束；数量与亮度保持克制，不遮挡主要文字，不改变倒计时启动时点，并支持 `prefers-reduced-motion` 降级。
- 后续主祭台 hover 目标调整为只保留一个信息框：烧香中或休息中优先保留带剩余时间的计时卡，去掉重复的香炉 metadata 浮层，避免两个卡片显示相同香数和进度。
- 符箓 hover 目标调整为更大幅度放大，优先保证符箓文字可读；允许临时遮蔽其他祭台内容，但不能改变点击入口、计时状态或复盘流程。
- 仪式台和复盘页提供“放弃本轮”保护入口。
- 仪式台和复盘页刷新/关闭页面前给出浏览器提示。
- 填写页已有草稿时，刷新/关闭页面前给出浏览器提示。
- 当前轮次在 Web 环境下暂存到 localStorage，在 Tauri 环境下暂存到桌面 JSON，刷新或重启后提示恢复或丢弃。
- 开发环境设置面板提供恢复场景 fixture，用于手动验收刷新恢复边界。
- 历史记录当前只用于回看、分析和反思，不提供一键复用。
- 提供 Tauri 桌面外壳，启动和小窗展开后的完整窗口均按 `960px × 760px` 外框目标验证，小窗按 `390px × 620px` 收起目标验证。
- Tauri 桌面版提供菜单栏 template icon 入口，用于隐藏和重新打开窗口；尺寸模式切换由小窗香炉入口和“保留并收起”负责。
- Tauri 小窗香炉舞台通过窗口位置会话提供独立透明拖拽热区；香炉炉身和线香区域在指针移动超过阈值后也会复用窗口位置会话移动小窗，并抑制随后一次点击，避免拖动手势误触展开。两个和三个香炉时的横排间距已收紧为 `6px`，小窗弱状态香炉不透明度已提高；本轮交互和视觉微调已由用户人工验证通过。
- Tauri 桌面版提供系统通知，用于专注和休息计时结束提醒。
- Tauri 桌面版通过系统文件对话框导入和导出历史 JSON。
- Tauri 桌面版提供“窗口置顶”和“在 Dock 中显示”设置。
- Tauri 桌面配置通过 `npm run check:desktop-config` 做静态回归检查。
- Tauri 桌面 JSON 损坏恢复通过 `npm run check:tauri-persistence-recovery` 做隔离原生冒烟：脚本复制并记录实际桌面 JSON，使用临时 `HOME` 下的损坏 fixture 启动精确 debug 二进制，确认损坏文件保留、安全 JSON 重写、恢复 toast 和实际桌面 JSON 逐字节不变。

## 当前不做

- 不做香炉开盖动画。
- 不做高密度符箓粒子、持续掉落碎纸或额外业务分支；允许在约 2 秒启动窗口内增加少量、局部、短促的小火苗，但不扩展为持续燃烧或高负载粒子系统。
- 不做复杂线香燃烧、香灰掉落或高频火星闪动；当前主祭台只保留低频火星呼吸。
- 不做高负载烟雾粒子或大幅烟雾位移；主祭台和小窗都只允许使用克制、低频、低干扰的烟雾反馈，使其在正常观看距离下可辨认。
- 不做复杂音效。
- 不接外部下载音频、持续鸣响或刺耳警报音。
- 不接后端。
- 不做 DMG、签名、公证和发布。
- 不接全局快捷键。
- 不做云同步。
- 不重新绘制复杂符箓、香炉、线香。
- 不在当前阶段继续制作第二轮 Q 版香炉、符箓或线香素材。
- 不导入本轮范围外的正式视觉素材。
- 不制作通知图标或完整发布 iconset。
- 不在正式构建中显示恢复场景 fixture。
- 不宣称宗教、玄学或超自然功效。

## 当前计时策略

倒计时以时间戳为准。运行中保存当前计时段的 `startedAt` 和 `durationSeconds`，刷新恢复时根据真实经过时间重新计算剩余秒数。旧版本暂存数据没有计时段字段时，恢复后会用保存的剩余秒数重新建立计时段。

## 优先级原则

完整业务流程和状态可靠性仍是不可回退的基线。当前核心闭环、桌面持久化、小窗、菜单栏入口和备份恢复已经可用，后续改动不得破坏计时、恢复、历史、复盘和通知时点。

视觉和声音不是发布前装饰，而是本自用应用仪式感的核心组成部分。仪式台视觉已使用正式 v1 素材，后续迭代继续通过既有插槽、manifest 和分层组件演进，不把复杂视觉状态烘焙进图片。

## 后续开发优先级

后续需求按以下顺序推进：

1. **真实自用暴露的小窗和祭台体验修正**：在既有小窗与可读性修正基础上，新增符箓燃烧小火苗精修。小火苗应沿局部起燃点和焦边短促摆动并随燃烧前缘推进，优先复用 `TalismanVisual` 现有纯视觉锚点和 CSS，不重做素材族，不新增业务状态，不改变约 2 秒燃烧契约、计时状态机、复盘流程、历史结构或桌面发布流程。
2. **Demo 展示路径保持回归**：完成上述修正后，固定检查填写页、完整主祭台、首次点燃及小火苗中间态、烧香中、休息、完成祭台、进入复盘、保存后历史存在、小窗收起拖拽、拖动位置保持、香炉 hover 剩余时间、香炉点击展开、小窗烟雾、主祭台单信息框、符箓大 hover 和声音开关路径。只修复实际暴露的视觉层级、动画节奏、声音时点和交互回退。
3. **正式升长期自用基线前完整验收**：新体验缺口和 Demo 路径回归完成后，再围绕当前候选链路做一次完整验收，决定是否升级长期自用基线。自动检查至少包含 `git diff --check`、`npm run check:release-self-use`、`npm run check:tauri-window-roundtrip -- --no-screenshots` 和 `npm run tauri:build`；人工验收按 `docs/SELF_USE_REGRESSION_RUNBOOK.md` 走打包 `.app` 核心路径，重点覆盖完成态恢复、进入复盘、保存复盘、重开后历史存在、小窗收起/展开、拖动位置保持和真实数据不被污染。本阶段不安装或覆盖日常 `.app`，不生成 DMG，不签名，不公证；只有验收无阻断后，才更新 `docs/SELF_USE_RELEASE_LOG.md` 并进入是否替换日常 `.app` 的单独决策。
4. **数据安全和日常自用体验**：完成前三项后，再继续强化完整备份、恢复、损坏 JSON 保护、历史筛选统计、复盘质量和常用任务体验。所有数据相关改动优先保护真实桌面 JSON，不把测试 fixture 写入日常数据。
5. **后续桌面便利能力**：再评估登录后自动启动、更完整后台策略和全局快捷键。全局快捷键当前不作为近期目标。
6. **正式发布准备**：最后再处理 DMG、签名、公证、面向他人的安装引导、完整发布 iconset 和公开发布验收。

通知横幅仍可能显示旧红色圆形占位图。该问题不影响通知正文和触发时机，继续保持最低优先级；仅在正式发布准备或用户主动重新提升优先级时继续排查。

## 书写原则

执行意图不是模板填充，而是用户重新确认行动承诺的过程。当前阶段保留手写或打字的摩擦感，不从历史记录一键复用旧意图；历史记录先服务于回看、分析和反思。
