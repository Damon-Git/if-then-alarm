# 自用回归 Runbook

本文用于在准备长期自用前，完整跑一遍当前版本。它不是公开发布测试，也不覆盖签名、公证、DMG、正式图标或 App Store 分发。

建议每次大迭代后按顺序执行。发现问题时先记录，不要边跑边改；跑完后再按优先级修。

## 一、准备

### 自动检查

先跑基础自动检查：

```bash
npm run check:release-self-use
npm run release:self-use-summary
```

如果要检查 Tauri 桌面运行：

```bash
npm run tauri:dev
```

如果本地 Playwright 和开发服务可用，再跑小窗回归：

```bash
npm run dev -- --host 127.0.0.1
npm run check:compact
```

### 测试数据

建议准备 3 套执行意图：

- 第 1 套：1 炷香，有 0 条预防性符箓。
- 第 2 套：2 炷香，有 1 条预防性符箓。
- 第 3 套：3 炷香，有 2-3 条预防性符箓。

示例文本：

- 情境性：当我坐到书桌前，就开始写今天的第一段文稿。
- 预防性：如果我想刷短视频，那么我先闭眼休息 5 分钟。

## 二、主流程

- [ ] 启动后进入填写页，默认显示 1 套执行意图。
- [ ] 能添加到最多 3 套，不能超过 3 套。
- [ ] 情境性执行意图校验有效：缺少“当”或“就”不能进入仪式台。
- [ ] 预防性执行意图校验有效：填写后缺少“如果”或“那么”不能进入仪式台。
- [ ] 点击“进入仪式台”后进入完整窗口主祭台，不直接进入小窗。
- [ ] 主祭台显示的香炉数量等于创建的套数。
- [ ] 1 / 2 / 3 炷香分别显示 1 / 2 / 3 根线香。
- [ ] 点击情境性符箓后先出现开始确认弹窗。
- [ ] 点击“开始这一套”后，情境性符箓先燃烧淡出约 2 秒，动画结束后第一炷香开始计时。
- [ ] 后续续香不再播放情境性符箓燃烧动画。

## 三、计时与完成

- [ ] 单炷香结束时弹出“这一炷香已经烧完。”。
- [ ] 有下一炷香时，“立刻续香”能进入下一炷香倒计时。
- [ ] 有下一炷香时，“休息 5 分钟”能进入休息倒计时。
- [ ] 休息结束时弹出“休息结束，是否继续下一炷香？”。
- [ ] 点击“开始下一炷香”后进入下一炷香倒计时。
- [ ] 每套全部线香烧完后，香炉闭盖并弱化。
- [ ] 每套全部线香烧完后，该套预防性符箓自动退场。
- [ ] 所有套组完成后停留在完成祭台场景，不自动进入复盘页。
- [ ] 完整窗口完成态显示“本轮香尽”和“进入复盘”入口。
- [ ] 点击“进入复盘”后才进入复盘页。

## 四、小窗

- [ ] 只有通过关闭确认里的“保留并收起”进入小窗。
- [ ] 小窗只显示用户创建的 1-3 个 Q 版香炉，不显示标题、符箓、历史、设置、复盘或灰色背景面板。
- [ ] 小窗香炉始终单行并排。
- [ ] 小窗香炉数量等于用户创建的套组数，不补空位。
- [ ] 小窗点击香炉只尝试展开完整窗口，不启动、不续香、不进入复盘。
- [ ] 小窗中最后一炷香结束后只显示完成态香炉，不直接弹复盘页。
- [ ] 小窗完成态点击香炉后回到完整窗口，仍停留完成祭台场景，由用户点击“进入复盘”。

## 五、主祭台视觉

- [ ] 三个香炉始终在同一条水平线上，不因状态变化上下跳动。
- [ ] 主祭台默认不常驻显示“未开始”“已完成”“3 炷香”“线香进度”等说明文字。
- [ ] 只有鼠标悬停到香炉命中区时，才显示清晰的香炉信息卡。
- [ ] 鼠标悬停到情境性符箓时，只放大符箓，不显示香炉状态信息。
- [ ] 鼠标悬停到预防性符箓时，只放大符箓，不显示香炉状态信息。
- [ ] 鼠标悬停到线香区域时，不显示香炉状态信息。
- [ ] 情境性符箓放大后不被香炉、线香或其他符箓遮挡。
- [ ] 预防性符箓放大后不被香炉、线香、桌面边界或其他符箓遮挡。
- [ ] 完成态闭盖香炉不显示线香穿出盖子。

## 六、复盘与历史

- [ ] 复盘页显示本轮摘要：套组数、总香数和预防性符箓数。
- [ ] 复盘页主路径突出“本次完成度”和“一句复盘”。
- [ ] “主要阻碍”和“下次调整”作为补充记录收起。
- [ ] 一句复盘为空时不能保存。
- [ ] 保存复盘后返回填写页。
- [ ] 保存后历史记录出现新记录。
- [ ] 历史摘要能清楚看到时间、完成度、套组数、总香数、约专注时长和一句复盘。
- [ ] 展开历史后能看到每套情境性执行意图、预防性执行意图和香数。
- [ ] 刷新或重启后历史记录仍存在。
- [ ] 历史记录不提供“再次使用”或一键复用入口。

## 七、桌面能力

- [ ] 系统通知在专注倒计时结束时触发，不在专注开始时触发。
- [ ] 系统通知在休息倒计时结束时触发，不在休息开始时触发。
- [ ] 小窗中本轮全部完成时，系统通知提示点击香炉回完整窗口复盘。
- [ ] 声音提醒关闭时不播放钟声。
- [ ] 声音提醒开启后，专注结束、休息结束和整轮完成各最多播放一次钟声。
- [ ] “窗口置顶”设置能生效并持久化。
- [ ] “在 Dock 中显示”设置能生效并持久化。
- [ ] 设置页“版本信息”能看到应用版本、构建类型、Bundle ID 和桌面数据版本。
- [ ] 历史导出打开 macOS 保存面板，并能写出 JSON。
- [ ] 历史导入打开 macOS 文件面板，并能导入本应用导出的 JSON。
- [ ] 设置页“导出完整备份”打开 macOS 保存面板，并能写出完整备份 JSON。
- [ ] 设置页“导入完整备份”打开 macOS 文件面板，导入前先弹确认。
- [ ] 确认导入完整备份后，历史、未完成轮次和设置会被替换；如果备份中有未完成轮次，应显示恢复提示。

## 八、恢复与保护

- [ ] 填写页有草稿时关闭窗口，会先确认。
- [ ] 仪式台或复盘页关闭窗口，会先确认。
- [ ] “放弃并退出”结束本轮，不写入历史。
- [ ] “保留并收起”进入小窗，计时继续。
- [ ] 进行中轮次重启后提示恢复或丢弃。
- [ ] 恢复后倒计时按真实经过时间继续或进入正确的结束弹窗。
- [ ] 保存复盘后重启，不再提示恢复旧轮次。
- [ ] 桌面本地数据文件异常时，应用不应无法启动。

## 九、问题记录

发现问题时按下面格式记录：

```text
是否影响长期自用：
```

## 十、通过标准

可以进入下一轮视觉或资产迭代的条件：

- 自动检查全部通过。
- 主流程、小窗、复盘历史、恢复保护无阻断问题。
- 准备用作日常自用版本时，已更新 `SELF_USE_RELEASE_LOG.md`。
- 不存在会丢失当前轮次、丢失历史、错误进入复盘或错误关闭应用的问题。
- 视觉问题只剩轻微对齐、素材精细度或非阻断体验问题。
