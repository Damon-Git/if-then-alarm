# 自用版回归 Runbook

本文用于准备长期自用版本前，按固定路径检查当前 `.app` 是否可以稳定使用。它不是公开发布测试，也不覆盖签名、公证、DMG 或 App Store 分发。

## 使用方式

- 小改动：跑“最短核心路径”即可。
- 涉及 Tauri、窗口、小窗、持久化、通知、声音、历史导入导出：跑“完整桌面路径”。
- 涉及主祭台、符箓、香炉、线香、动画：额外跑“视觉状态路径”。
- 准备替换日常使用的 `.app`：自动检查、最短核心路径、完整桌面路径都要通过，并更新 `SELF_USE_RELEASE_LOG.md`。

发现问题时先记录，不要边跑边改；跑完后再按严重程度处理。

## 一、自动检查

每次准备自用版前先执行：

```bash
npm run check:release-self-use
npm run release:self-use-summary
```

如果本轮替换或新增视觉素材，可以单独先跑：

```bash
npm run check:visual-assets
```

如果本轮替换或新增钟声素材，可以单独先跑：

```bash
npm run check:sound-assets
```

如果本轮改过小窗布局或响应式样式，并且本地开发服务可用，再执行：

```bash
npm run dev -- --host 127.0.0.1
npm run check:compact
```

如果本轮准备验证桌面运行：

```bash
npm run tauri:dev
```

如果本轮改过 Tauri 窗口外壳、尺寸切换或小窗拖拽，并且当前没有正在运行的 debug 或 release 进程，可以先执行隔离的原生窗口往返冒烟：

```bash
npm run check:tauri-window-roundtrip
```

脚本会备份实际桌面 JSON，使用临时 `HOME` 启动精确 debug 二进制，并把应用窗口截图和审计记录写入 ignored `artifacts/tauri-window-roundtrip/`。脚本不安装或覆盖 `.app`，不生成 DMG，也不替代人工桌面验收。

如果本轮准备生成新的 `.app`：

```bash
npm run tauri:build
```

## 二、测试数据

建议固定准备 3 套执行意图：

- 第 1 套：1 炷香，0 条预防性符箓。
- 第 2 套：2 炷香，1 条预防性符箓。
- 第 3 套：3 炷香，2-3 条预防性符箓。

示例文本：

- 情境性：当我坐到书桌前，就开始写今天的第一段文稿。
- 预防性：如果我想刷短视频，那么我先闭眼休息 5 分钟。

## 三、最短核心路径

这组用于判断“今天能不能继续自己长期使用”。

- [ ] 启动后进入填写页，默认显示 1 套执行意图。
- [ ] 能添加到最多 3 套，不能超过 3 套。
- [ ] 情境性执行意图校验有效：缺少“当”或“就”不能进入仪式台。
- [ ] 预防性执行意图校验有效：填写后缺少“如果”或“那么”不能进入仪式台。
- [ ] 点击“进入仪式台”后进入完整窗口主祭台，不直接进入小窗。
- [ ] 主祭台显示的香炉数量等于创建的套数。
- [ ] 1 / 2 / 3 炷香分别显示 1 / 2 / 3 根线香。
- [ ] 点击情境性符箓后先出现开始确认弹窗。
- [ ] 点击“开始这一套”后，情境性符箓先播放约 2 秒分层燃烧退场，动画结束后第一炷香开始计时。
- [ ] 后续续香不再播放情境性符箓燃烧动画。
- [ ] 单炷香结束时弹出“这一炷香已经烧完。”。
- [ ] 有下一炷香时，“立刻续香”能进入下一炷香倒计时。
- [ ] 有下一炷香时，“休息 5 分钟”能进入休息倒计时。
- [ ] 休息结束时弹出“休息结束，是否继续下一炷香？”。
- [ ] 每套全部线香烧完后，香炉闭盖并弱化。
- [ ] 每套全部线香烧完后，该套预防性符箓自动退场。
- [ ] 所有套组完成后停留在完成祭台场景，不自动进入复盘页。
- [ ] 完整窗口完成态显示“本轮香尽”和“进入复盘”入口。
- [ ] 点击“进入复盘”后才进入复盘页。
- [ ] 一句复盘为空时不能保存。
- [ ] 保存复盘后返回填写页。
- [ ] 保存后历史记录出现新记录。
- [ ] 刷新或重启后历史记录仍存在。

## 四、完整桌面路径

这组用于验证 Tauri 安装版和桌面能力。

- [ ] 内部测试 `.app` 可以打开，初始窗口接近完整窗口尺寸。
- [ ] 点击“进入仪式台”后仍在完整窗口，不自动进入小窗。
- [ ] 手动拖小完整窗口时，不自动切换成小窗。
- [ ] 点击窗口关闭按钮时弹出应用内确认弹窗。
- [ ] “放弃并退出”结束本轮，不写入历史。
- [ ] “保留并收起”进入小窗，计时继续。
- [ ] 小窗只显示用户创建的 1-3 个 Q 版香炉，不显示标题、符箓、历史、设置、复盘或背景面板。
- [ ] 小窗香炉始终单行并排。
- [ ] 小窗香炉横排上方拖拽热区可以可靠移动窗口。
- [ ] 从拖拽热区开始拖动时，不会误触香炉展开完整窗口。
- [ ] 从香炉炉身或线香区域按下后明显移动时，小窗会跟随鼠标移动，并且不会在松手后误触展开；正常单击仍可展开完整窗口。
- [ ] 小窗点击香炉只尝试展开完整窗口，不启动、不续香、不进入复盘。
- [ ] 小窗点击香炉恢复完整窗口后，原生外框仍接近 `960px × 760px`，不会叠加标题栏高度。
- [ ] 小窗中最后一炷香结束后只显示完成态香炉，不直接弹复盘页。
- [ ] 小窗完成态点击香炉后回到完整窗口，仍停留完成祭台场景，由用户点击“进入复盘”。
- [ ] 系统通知在专注倒计时结束时触发，不在专注开始时触发。
- [ ] 系统通知在休息倒计时结束时触发，不在休息开始时触发。
- [ ] 小窗中本轮全部完成时，系统通知提示点击香炉回完整窗口复盘。
- [ ] 声音提醒关闭时不播放钟声。
- [ ] 声音提醒开启后，单炷结束播放单击钟声，休息结束播放双击钟声，整轮完成播放三段收束钟声；每个事件最多播放一次。
- [ ] 打开“放弃本轮”确认框并停留超过当前剩余时间时，不会提前播放钟声；取消放弃后按剩余时间重新提醒。
- [ ] “窗口置顶”设置能生效并持久化。
- [ ] “在 Dock 中显示”设置能生效并持久化。
- [ ] 设置页“版本信息”能看到应用版本、构建类型、Bundle ID 和桌面数据版本。
- [ ] 历史导出打开 macOS 保存面板，并能写出 JSON。
- [ ] 历史导入打开 macOS 文件面板，并能导入本应用导出的 JSON。
- [ ] 设置页“导出完整备份”打开 macOS 保存面板，并能写出完整备份 JSON。
- [ ] 设置页“导入完整备份”打开 macOS 文件面板，导入前先弹确认。
- [ ] 确认导入完整备份后，历史、未完成轮次和设置会被替换；如果备份中有未完成轮次，应显示恢复提示。
- [ ] 进行中轮次重启后提示恢复或丢弃。
- [ ] 恢复后倒计时按真实经过时间继续，或进入正确的结束弹窗。
- [ ] 保存复盘后重启，不再提示恢复旧轮次。
- [ ] 桌面本地数据文件异常时，应用不应无法启动。

## 五、视觉状态路径

这组用于验证主祭台、符箓、香炉和线香状态。

- [ ] 三个香炉始终在同一条水平线上，不因状态变化上下跳动。
- [ ] 主祭台默认不常驻显示“未开始”“已完成”“3 炷香”“线香进度”等说明文字。
- [ ] 只有鼠标悬停到香炉命中区时，才显示清晰的香炉信息卡。
- [ ] 烧香中或休息中悬停香炉时，计时卡与香炉信息卡同时清晰可读，彼此不重叠，也不遮挡线香、香炉主体或预防性符箓。
- [ ] 鼠标悬停到情境性符箓时，只放大符箓，不显示香炉状态信息。
- [ ] 鼠标悬停到预防性符箓时，只放大符箓，不显示香炉状态信息。
- [ ] 鼠标悬停到线香区域时，不显示香炉状态信息。
- [ ] 情境性符箓放大后不被香炉、线香或其他符箓遮挡。
- [ ] 预防性符箓放大后不被香炉、线香、桌面边界或其他符箓遮挡。
- [ ] 情境性执行意图文本位于符箓两侧文字栏，不压住中央符文。
- [ ] 预防性执行意图文本位于符箓两侧文字栏，不压住中央符文。
- [ ] 情境性符箓底部位于香炉上方，不与香炉和线香叠成一团。
- [ ] 预防性符箓贴近桌子前沿水平排列，2-3 张时不上下堆叠。
- [ ] 未开始、烧香中、休息中的香炉保持开盖。
- [ ] 完成态香炉闭盖并弱化，不显示线香穿出盖子。
- [ ] 完成态香炉合盖和完成卡片浮入短促克制，不出现明显跳动。
- [ ] 情境性符箓首次启动时，不是整张卡片统一淡出；应能辨认局部起燃、焦边推进、灰化和收束退场。
- [ ] 情境性符箓燃烧期间，模板和文字共同退场，不出现文字单独悬空。
- [ ] 主祭台当前正在燃烧的线香会随进度从顶部转为香灰。
- [ ] 主祭台烧香中火星低频呼吸，烟雾在正常观看距离下可辨认，并保持缓慢、不规则、低干扰漂移。
- [ ] 1-3 个香炉同时出现时，烟雾和火星不形成同步闪烁，不遮挡符箓文字、计时卡、香炉信息卡、香炉主体或相邻香炉。
- [ ] 完成态不残留持续烟雾；系统开启“减少动态效果”时，烧香中只保留弱化静态烟雾提示，休息余烟短促淡出。
- [ ] 已烧完的线香保持灰色香灰状态，不显示为完整待燃香。
- [ ] 未烧到的线香保持完整深色待燃状态。
- [ ] 休息中当前线香保持已烧完状态，只保留短暂余烟和冷却过渡，不继续表现为燃烧。
- [ ] 系统开启“减少动态效果”时，主祭台新增动画立即降级，最终状态仍正确。

## 七、通过标准

可以进入下一轮视觉或资产迭代的条件：

- 自动检查通过。
- 最短核心路径无阻断问题。
- 如果准备替换日常使用 `.app`，完整桌面路径无阻断问题。
- 如果本轮改过视觉状态，视觉状态路径无阻断问题。
- 准备用作日常自用版本时，已更新 `SELF_USE_RELEASE_LOG.md`。
- 不存在会丢失当前轮次、丢失历史、错误进入复盘或错误关闭应用的问题。
- 视觉问题只剩轻微对齐、素材精细度或非阻断体验问题。
