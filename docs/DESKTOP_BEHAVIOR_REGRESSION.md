# 桌面行为回归清单

本文用于每次修改 Tauri 外壳、桌面持久化、系统通知、窗口行为、导入导出后做组合验收。它不替代 `docs/TAURI_DESKTOP_CHECKLIST.md`，而是按真实桌面使用顺序串起高风险交叉点。

## 目标

- 固定当前桌面版行为基线，避免配置、权限或入口被后续改动误删。
- 优先覆盖小窗桌面使用中最容易互相影响的功能：关闭恢复、通知、Dock、菜单栏、置顶、导入导出、桌面 JSON。
- 不要求每次都验收视觉细节；视觉细节仍以对应的 UI 验收清单为准。

## 自动静态检查

修改桌面相关代码后，先执行：

```bash
npm run check:desktop-config
npm run check:compact
```

这个检查不启动 Tauri GUI，不读取用户真实数据，只校验项目内的关键配置和代码入口：

- Tauri identifier 仍为 `com.damon.jijirululing`。
- 初始窗口仍为 `960 × 760`。
- 小窗模式仍为 `390 × 620`。
- 完整窗口尺寸仍为 `960 × 760`。
- 桌面权限包含 Dock、窗口隐藏、窗口置顶、窗口装饰切换、窗口阴影切换、标题栏样式切换、窗口背景色切换、小窗拖拽、系统通知、系统文件对话框。
- Rust 依赖包含通知插件、对话框插件和 `tray-icon` feature。
- 桌面持久化文件名仍为 `persistence.v1.json`。
- 关键适配器、命令和文档仍存在。

如果本轮修改 Tauri 窗口外壳、尺寸切换或小窗拖拽，再在 macOS 内部开发环境显式执行：

```bash
npm run check:tauri-window-roundtrip
```

这个原生冒烟会先备份实际桌面 JSON，拒绝复用已有 debug 或 release 进程，再使用临时 `HOME` 启动精确 debug 二进制。它会自动验证初始完整窗口、两次 `390 × 620 → 960 × 760` 往返、独立拖拽热区和香炉炉身区域移动后尺寸不变且不误触展开，并在结束后确认实际桌面 JSON 逐字节不变。应用窗口截图和审计记录只写入 ignored `artifacts/tauri-window-roundtrip/`。原生标题栏、阴影、透明底板观感以及炉身和线香区域拖动手感仍保留人工验收。

如果本轮修改桌面持久化，或准备长期自用基线，再显式执行：

```bash
npm run check:tauri-persistence-recovery
```

这个原生冒烟同样先备份实际桌面 JSON，拒绝复用已有 debug 或 release 进程，再使用临时 `HOME` 启动精确 debug 二进制。它会写入隔离损坏 fixture，确认 Rust 命令保留 `persistence.v1.corrupt-*.json`、前端初始化重写安全 JSON、实际桌面 JSON 逐字节不变，并把应用窗口截图和审计记录只写入 ignored `artifacts/tauri-persistence-recovery/`。按产品契约，恢复过程可能只读旧 WebView `localStorage` 作为迁移来源；审计记录只保留恢复摘要，不复制恢复后的完整内容。恢复 toast 仍保留截图人工抽验。

`check:compact` 需要先启动 Web 开发服务。它用于防止小窗语义回退：

- 点击“开始创造”后仍进入完整主祭台，不自动切到小窗。
- 进入完整主祭台后所有任务仍保持未开始，不自动开始计时、不弹开始确认、不进入复盘。
- 完整主祭台使用 `stage` 香炉素材族，不混入 `compact` 小窗素材族。
- 手动窄视口仍显示完整仪式台，不自动进入小窗。
- 显式小窗模式下只显示香炉舞台。
- 小窗 ritual 状态隐藏标题、历史、设置、仪式台标题和主祭台。
- 小窗 ritual 状态不显示符箓、计时面板或复盘面板。
- 小窗香炉单行并排，数量与任务数量一致。
- 小窗香炉横排上方存在使用 Tauri 窗口位置会话的独立拖拽热区，热区与香炉点击入口不重叠。
- 浏览器调试环境中按下拖拽热区不会修改业务状态；从香炉按钮自身按下后明显移动会进入窗口移动语义并抑制随后一次展开点击；桌面窗口实际移动由隔离原生冒烟和 Tauri 人工回归共同覆盖。
- 小窗 ritual 状态下 `html`、`body`、`#root`、应用主壳、面板、香炉舞台和香炉按钮都不绘制背景。
- 小窗中正在烧香的香炉比未开始香炉更明显，并显示克制烟雾动画，但仍不绘制背景面板。
- 小窗香炉不常驻显示状态文案、倒计时、提示或长执行意图；hover/focus 时必须临时显示剩余时间，且浮层完整位于小窗视口和香炉按钮内部垂直范围，不得被裁剪。
- 点击小窗香炉只尝试回到完整窗口，不打开开始确认、不启动计时、不续香、不进入复盘。
- 小窗中最后一炷香结束后不直接进入复盘；完整窗口展开失败时仍停留小窗完成态。

## 手动回归顺序

### 1. 内部打包冒烟

- 执行 `npm run tauri:build`。
- 确认生成 `src-tauri/target/release/bundle/macos/急急如律令.app`。
- 确认 `.app/Contents/MacOS/jiji-rululing`、`.app/Contents/Info.plist` 和 `.app/Contents/Resources/急急如律令.icns` 存在。
- 确认 `Info.plist` 中 bundle identifier 为 `com.damon.jijirululing`，版本为 `0.2.0`。
- 双击或右键打开 `.app` 后能启动窗口。
- 打包版完成一轮复盘后重新打开，历史记录仍可查看。

### 2. 启动与入口

- 执行 `npm run tauri:dev`。
- 桌面窗口能启动，标题为“急急如律令”。
- 初始窗口接近 `960 × 760`，没有横向滚动条。
- macOS 菜单栏右侧能看到简化香炉 template icon。
- 点击 template icon 可以隐藏当前窗口，再次点击可以按隐藏前的尺寸模式重新显示并获得焦点。
- 手动把完整窗口拖窄后，仍显示完整仪式台，不自动切换为小窗香炉舞台。
- 点击“开始创造”后，先显示完整主祭台；只有确认“保留并收起”后才进入小窗香炉舞台。
- 小窗拖拽热区可以可靠移动窗口；从热区开始拖动时不展开完整窗口；从香炉炉身或线香区域开始的明显移动也可以移动窗口且不会误触展开。
- 小窗拖动到新位置后，点击香炉展开完整窗口，再选择“保留并收起”时，小窗回到上一次移动后的位置。
- 小窗烧香中 hover/focus 当前香炉会完整显示剩余时间，浮层不越过透明内容区边界，且烟雾动画可见但克制。
- 单击任意小窗香炉仍只展开完整窗口，不启动、不续香、不进入复盘。
- 从小窗展开后，完整窗口外框仍接近 `960 × 760`，不会叠加原生标题栏高度。

### 3. 桌面数据

- 新建一轮并保存复盘，历史记录能出现新记录。
- 重启后历史、设置、未完成轮次恢复行为符合预期。
- macOS 开发环境下，桌面 JSON 可在 app data 目录中找到，文件名为 `persistence.v1.json`。
- 修改“窗口置顶”和“在 Dock 中显示”后重启，设置仍保留。

### 4. 窗口行为组合

- 开启“窗口置顶”后，窗口保持在其他窗口上方。
- 关闭“窗口置顶”后，窗口恢复普通层级。
- 关闭“在 Dock 中显示”后，Dock 图标消失，但菜单栏 template icon 入口仍能打开窗口。
- 重新开启“在 Dock 中显示”后，Dock 图标恢复。
- 在 Dock 隐藏状态下关闭、收起到小窗或隐藏窗口，不应失去重新打开入口。

### 5. 关闭与恢复

- 填写页有草稿时点击窗口关闭按钮，会出现确认弹窗。
- 草稿关闭确认中选择继续填写，窗口保持打开。
- 草稿关闭确认中选择关闭窗口，重新启动后不恢复草稿。
- 仪式台进行中点击关闭按钮，会出现确认弹窗。
- 进行中关闭确认中点击弹窗关闭按钮，窗口保持打开，当前轮次不变化。
- 进行中关闭确认中选择“放弃并退出”，窗口关闭，重新启动后不恢复本轮且不写入历史。
- 进行中关闭确认中选择“保留并收起”，窗口切回小窗模式且计时继续。
- 重新启动后，如果有未完成轮次，会出现恢复提示。
- 保存复盘后重新启动，不再提示恢复旧轮次。

### 6. 通知时点

- 一炷香计时开始时，不应立刻弹出“这一炷香已经烧完。”通知。
- 一炷香倒计时结束时，系统通知显示“这一炷香已经烧完。”。
- 休息计时开始时，不应立刻弹出“休息结束，是否继续下一炷香？”通知。
- 休息倒计时结束时，系统通知显示“休息结束，是否继续下一炷香？”。
- 开始下一段计时后，不会继续保留上一段的待触发通知。
- 放弃本轮、保存复盘或丢弃恢复轮次后，不会继续触发旧通知。

### 7. 声音提醒

- “声音提醒”默认关闭。
- 关闭声音提醒时，一炷香结束、休息结束和整轮完成都不播放声音。
- 开启声音提醒后，单炷结束播放单击钟声，休息结束播放双击钟声，整轮完成播放三段收束钟声；每个事件最多播放一次。
- 一炷香计时开始时，不播放声音。
- 休息计时开始时，不播放声音。
- 开始下一段计时后，不会继续保留上一段的待触发声音。
- 打开“放弃本轮”确认框并停留超过当前剩余时间时，不会提前播放钟声；取消放弃后按剩余时间重新提醒。
- 放弃本轮、保存复盘或丢弃恢复轮次后，不会继续触发旧声音。
- 关闭系统通知权限时，声音提醒仍按设置独立工作；系统静音、勿扰和系统音量以 macOS 行为为准。

### 8. 历史导入导出与完整备份

- 点击“导出历史”会打开 macOS 保存面板。
- 取消保存面板后，不显示导出成功 toast。
- 选择保存路径后，会写入本应用历史 JSON。
- 点击“导入历史”会打开 macOS 打开文件面板。
- 取消打开文件面板后，不显示导入失败 toast。
- 选择本应用导出的 JSON 后，能完成导入并显示新增或跳过数量。
- 导入导出过程中开启窗口置顶时，系统对话框仍可正常操作。
- 点击“导出完整备份”会打开 macOS 保存面板，写出包含历史、当前未完成轮次和设置的 JSON。
- 点击“导入完整备份”会打开 macOS 打开文件面板，选择文件后先显示确认弹窗。
- 确认导入完整备份后，当前历史、未完成轮次和设置被替换；如果备份中有未完成轮次，应显示恢复提示。
- 选择历史导出文件导入完整备份时，应显示失败提示，不覆盖当前数据。

### 9. 高风险交叉场景

- Dock 隐藏 + 收起到小窗 + 菜单栏隐藏/重新打开。
- 窗口置顶 + 系统保存面板。
- 进行中轮次 + 收起到小窗 + 计时结束通知。
- 进行中轮次 + 收起到小窗 + 声音提醒开启 + 计时结束。
- 进行中轮次 + 重启恢复 + 继续下一炷香。
- 桌面 JSON 损坏备份 + 自动恢复提示。

## 当前不覆盖

- 不覆盖全局快捷键。
- 不覆盖 DMG、签名、公证和发布。
- 不覆盖符箓、香炉、线香正式视觉素材和动画。
