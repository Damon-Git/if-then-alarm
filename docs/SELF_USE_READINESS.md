# 自用版稳定性基线

本文用于判断当前版本是否达到“可长期自己使用的 macOS 桌面小工具”基线。它不代表正式发布标准，也不覆盖签名、公证、DMG、正式图标或公开试用。

## 目标

- 主流程可以稳定完成一轮专注、休息、续香和复盘。
- macOS 桌面窗口、小窗、菜单栏入口、关闭恢复和桌面持久化不互相打架。
- 小窗只承担低干扰计时器职责，完整操作回到完整窗口处理。
- 历史、当前未完成轮次和设置可以长期保留。
- 出错时优先保住当前轮次，不静默丢失用户正在进行的行动承诺。

## 自动检查

每次准备作为自用版继续使用前，先跑：

```bash
npm run check:release-self-use
npm run release:self-use-summary
```

`npm run check:release-self-use` 会串行执行 `npm run test`、`npm run build`、`npm run check:desktop-config` 和 `npm run check:self-use`，并先确认前端版本与 Tauri 版本一致。`npm run release:self-use-summary` 会打印当前版本、Git 提交、Bundle ID、数据版本和构建产物路径，方便写入 `SELF_USE_RELEASE_LOG.md`。

如果本地 Playwright 和开发服务可用，再跑小窗图形回归：

```bash
npm run dev -- --host 127.0.0.1
npm run check:compact
```

`npm run check:self-use` 是静态基线检查，只确认长期自用所需的关键脚本、文档和代码入口没有被误删。它会被 `npm run check:release-self-use` 调用，但不替代 Tauri 真机运行或小窗截图检查。

更完整的手动回归路径见 [SELF_USE_REGRESSION_RUNBOOK.md](SELF_USE_REGRESSION_RUNBOOK.md)。准备把某个版本作为日常自用版本前，按该清单完整走一遍。

内部 `.app` 安装、替换、数据备份和恢复见 [MACOS_SELF_USE_INSTALL.md](MACOS_SELF_USE_INSTALL.md)。准备把构建产物放进日常使用位置前，按该文档检查。

准备把某个 `.app` 作为新的日常自用版本前，还要在 [SELF_USE_RELEASE_LOG.md](SELF_USE_RELEASE_LOG.md) 记录版本信息、Git 提交、数据版本、验收结果、完整备份和回滚方式。设置页会显示当前应用版本、构建类型、Bundle ID 和桌面数据版本，用于安装后核对。

## 桌面冒烟

需要确认内部 `.app` 能正常生成和启动：

```bash
npm run tauri:build
```

生成后检查：

- `src-tauri/target/release/bundle/macos/急急如律令.app` 存在。
- 如果替换日常自用 `.app`，先退出正在运行的旧应用。
- 双击或右键打开 `.app` 能启动完整窗口。
- 初始窗口接近 `960 × 760`。
- 菜单栏右侧有简化香炉 template icon。
- 完成一轮复盘后关闭再打开，历史记录仍在。
- 数据文件位于 `~/Library/Application Support/com.damon.jijirululing/persistence.v1.json`，替换 `.app` 不应清空该文件。

## 必验路径

### 主流程

- 创建 1-3 套执行意图。
- 每套选择 1-3 炷香。
- 点击“进入仪式台”后先进入完整主祭台，不自动进入小窗。
- 点击情境性符箓后先弹开始确认。
- 确认后先播放情境符箓燃烧淡出动画，动画结束后开始倒计时。
- 单炷结束后可以休息或续香。
- 全部香结束后停留在完成态主祭台；用户主动点击“进入复盘”后进入复盘。
- 保存复盘后历史出现新记录。

### 小窗

- 只有“保留并收起”进入小窗。
- 小窗只显示用户创建的 1-3 个香炉。
- 香炉始终单行并排。
- 小窗不显示标题、历史、设置、符箓、计时文字、复盘或背景面板。
- 点击小窗香炉只尝试回到完整窗口，不启动、不续香、不放弃。
- 小窗中最后一炷香结束后，不直接弹复盘；点击完成态香炉回完整窗口后再复盘。

### 关闭与恢复

- 填写页有草稿时关闭，会先确认。
- 仪式台或复盘中关闭，会先确认。
- “放弃并退出”清空当前轮次，不写历史。
- “保留并收起”进入小窗，计时继续。
- 重启后未完成轮次会提示恢复或丢弃。
- 保存复盘后重启，不再提示恢复旧轮次。

### 桌面能力

- 系统通知在倒计时结束时触发，不在倒计时开始时触发。
- 声音提醒按设置开关工作。
- “窗口置顶”设置能生效并持久化。
- “在 Dock 中显示”设置能生效并持久化。
- 设置页“版本信息”显示当前应用版本、构建类型、Bundle ID 和数据版本，并与本次自用发布记录一致。
- 历史导出和导入能打开 macOS 文件面板。
- 完整备份导出和导入能打开 macOS 文件面板；导入前必须先确认。
- 桌面 JSON 损坏时不会让应用无法启动，应能备份或恢复到安全状态。

## 暂不纳入自用基线

- 正式通知图标。
- DMG、签名、公证和发布。
- 全局快捷键。
- 香炉开盖动画。
- 正式符箓燃烧素材和复杂燃烧粒子。
- 线香燃烧、烟雾、香灰动画。
- 面向他人试用的首次启动引导。

## 判断标准

如果自动检查通过，且上述必验路径没有阻断问题，可以认为当前版本达到“可长期自己使用”的阶段性基线。

如果出现以下问题，应先修复再继续做视觉：

- 当前轮次丢失或错误写入历史。
- 小窗和完整窗口状态互相串扰。
- 完成态提前进入复盘或无法回到复盘。
- 通知或声音在错误时间触发。
- 桌面持久化无法恢复历史、设置或未完成轮次。
