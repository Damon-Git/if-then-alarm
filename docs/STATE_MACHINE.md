# 状态流转

当前 MVP 使用页面阶段和执行意图套组状态两层状态。

## 页面阶段

`setup`：填写执行意图。

`ritual`：进入仪式台，用户可以点击尚未完成的情境性符箓开始计时。

`review`：全部执行意图套组完成后进入复盘。

## 执行意图套组状态

`idle`：尚未开始。可以点击情境性符箓开始第一炷香。

`burning`：当前炷香正在倒计时。倒计时结束后：

- 如果还有下一炷香，弹出“这一炷香已经烧完。”提醒。
- 如果没有下一炷香，进入 `completed`。

`resting`：用户选择休息后进入休息倒计时。休息结束后弹出“休息结束，是否继续下一炷香？”提醒。

`completed`：该套执行意图的全部香已经完成，不再允许重新开始。

## 关键动作

1. 提交填写页：`setup -> ritual`，所有套组初始化为 `idle`，当前香为第 1 炷。
2. 点击情境性符箓：目标套组 `idle -> burning`，倒计时使用 `DEV_TIMER_SECONDS`。
3. 单炷香结束且仍有下一炷：保持当前套组，弹出休息/续香选择。
4. 点击“立刻续香”：当前香序号加 1，进入 `burning`。
5. 点击“休息 5 分钟”：进入 `resting`，倒计时使用 `DEV_BREAK_SECONDS`。
6. 休息结束后点击“开始下一炷香”：当前香序号加 1，进入 `burning`。
7. 最后一炷香结束：目标套组进入 `completed`。
8. 所有套组均为 `completed`：`ritual -> review`。
9. 保存复盘：写入 localStorage，并返回 `setup`。
10. 放弃本轮：在 `ritual` 或 `review` 阶段弹出确认；确认后清空当前轮次状态，返回 `setup`，不写入历史。

## 并发约束

当前 MVP 一次只允许一个套组处于 `burning` 或 `resting`。这是为了让提醒、休息和续香流程保持明确。未来如果需要多套并行计时，可以把计时器状态移动到每个套组内部独立管理。

## 可用性保护

当前 MVP 不做完整 session 恢复，但在 `ritual` 和 `review` 阶段启用刷新/关闭页面保护。用户刷新或关闭页面时，浏览器会提示当前轮次尚未保存。

应用内“放弃本轮”使用确认弹窗。确认弹窗打开时，当前倒计时会暂停；用户取消后继续当前轮次，用户确认后返回填写页。
