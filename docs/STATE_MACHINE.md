# 状态流转

交互语义总览见 `docs/INTERACTION_MODEL.md`。本文只描述业务状态流转，不定义窗口形态、菜单栏入口或关闭恢复的完整产品语义。

当前 MVP 使用页面阶段和执行意图套组状态两层状态。

## 页面阶段

`setup`：填写执行意图。已有草稿时，刷新或关闭页面会触发浏览器保护提示。

`ritual`：进入仪式台，用户可以点击尚未完成的情境性符箓，确认后开始计时。

`review`：全部执行意图套组完成后，在完整窗口中进入复盘。

## 执行意图套组状态

`idle`：尚未开始。可以点击情境性符箓打开开始确认，确认后开始第一炷香。

`burning`：当前炷香正在倒计时。倒计时结束后：

- 如果还有下一炷香，弹出“这一炷香已经烧完。”提醒。
- 如果没有下一炷香，进入 `completed`。

`resting`：用户选择休息后进入休息倒计时。休息结束后弹出“休息结束，是否继续下一炷香？”提醒。

`completed`：该套执行意图的全部香已经完成，不再允许重新开始。

## 主祭台视觉语义

业务状态进入主祭台前，先由 `src/lib/visualState.ts` 的 `getStageIntentVisualSemantics` 映射为视觉语义。这样后续替换符箓、香炉、线香或动画时，不需要重新解释业务状态。

- `idle`：情境符箓可作为启动入口；预防性符箓可见；香炉开盖；线香完整待燃；不渲染计时面板。
- `burning`：情境符箓退场；预防性符箓可见；香炉开盖；当前线香按进度燃烧；渲染计时面板。
- `resting`：情境符箓保持退场；预防性符箓可见；香炉开盖；当前线香保持已烧完状态，不表现为继续燃烧；渲染休息计时面板。
- `completed`：情境符箓保持退场；预防性符箓退场；香炉闭盖并弱化；线香隐藏；不渲染计时面板。

主祭台的状态标签、香数和线香进度属于辅助信息，默认不常驻显示，只在鼠标悬停到香炉命中区时浮出。符箓 hover 只放大符箓，线香区域 hover 不触发辅助信息。

主祭台点击入口只保留在情境性符箓：点击后打开开始确认弹窗。预防性符箓、香炉和线香不推进业务状态；小窗香炉点击只尝试展开完整窗口，不启动或续香。

未来情境符箓燃烧动画只在第一次确认开始时播放一次，目标时长约 2 秒；动画完成后的状态仍然是情境符箓退场。同一套执行意图续香时不再播放该动画。

## 关键动作

1. 提交填写页：`setup -> ritual`，所有套组初始化为 `idle`，当前香为第 1 炷。
2. 点击情境性符箓：打开开始确认弹窗，展示本套执行意图、香数和当前计时模式。
3. 小窗香炉查看：小窗舞台中，点击任意香炉只会尝试展开并聚焦完整窗口，不改变业务状态，不打开开始确认。
4. 确认开始：目标套组 `idle -> burning`，倒计时使用当前计时模式的专注时长。
5. 取消开始：关闭确认弹窗，目标套组保持 `idle`。
6. 单炷香结束且仍有下一炷：保持当前套组，弹出休息/续香选择。
7. 点击“立刻续香”：当前香序号加 1，进入 `burning`。
8. 点击“休息 5 分钟”：进入 `resting`，倒计时使用当前计时模式的休息时长。
9. 休息结束后点击“开始下一炷香”：当前香序号加 1，进入 `burning`。
10. 最后一炷香结束：目标套组进入 `completed`。
11. 所有套组均为 `completed`：完整窗口中 `ritual -> review`；小窗中继续停留在 `ritual` 的完成香炉场景，用户点击小香炉展开完整窗口后再进入 `review`。
12. 保存复盘：写入 localStorage，并返回 `setup`。
12. 放弃本轮：在 `ritual` 或 `review` 阶段弹出确认；确认后清空当前轮次状态，返回 `setup`，不写入历史。
13. 恢复本轮：启动时如果发现暂存 session，用户可选择恢复到 `ritual` 或 `review`，也可丢弃该 session。

## 并发约束

当前 MVP 一次只允许一个套组处于 `burning` 或 `resting`。这是为了让提醒、休息和续香流程保持明确。未来如果需要多套并行计时，可以把计时器状态移动到每个套组内部独立管理。

## 可用性保护

当前 MVP 在 `ritual` 和 `review` 阶段启用刷新/关闭页面保护。用户刷新或关闭页面时，浏览器会提示当前轮次尚未保存。

填写页在已有草稿时也启用刷新/关闭页面保护。草稿只在当前页面内保留，不写入历史，也不自动从历史记录复用。

应用内“放弃本轮”使用确认弹窗。确认弹窗打开时，当前倒计时会冻结当前剩余秒数；用户取消后用该剩余秒数重新建立计时段，用户确认后返回填写页。

## Session 暂存

当前 MVP 会把未完成轮次暂存到 localStorage。暂存内容包括：

- 当前页面阶段：`ritual` 或 `review`
- 当前执行意图套组
- 当前倒计时剩余秒数
- 当前活动计时段：`startedAt` 和 `durationSeconds`
- 当前应用内提醒弹窗
- 当前计时模式
- 最近更新时间

倒计时不再依赖“每秒减 1”作为真实来源。运行中会保存当前计时段的开始时间和总时长，界面上的剩余秒数由当前时间推导。

刷新后，应用会提示用户恢复或丢弃未完成轮次。恢复时会根据 `startedAt`、`durationSeconds` 和当前时间重新计算剩余秒数：

- 如果专注计时仍未结束，恢复到 `burning` 并继续显示真实剩余时间。
- 如果专注计时已经结束且还有下一炷香，恢复后直接显示“这一炷香已经烧完。”。
- 如果专注计时已经结束且是最后一炷香，恢复后该套进入 `completed`；全部完成且处于完整窗口时进入 `review`。
- 如果休息计时已经结束，恢复后直接显示“休息结束，是否继续下一炷香？”。

旧版本暂存 session 如果没有活动计时段字段，恢复时仍会使用保存的剩余秒数重新建立计时段。保存复盘、确认放弃本轮、或丢弃恢复提示时，会清除暂存 session。

## 开发恢复场景

开发环境下，设置面板会显示“开发恢复场景”。这些按钮只用于手动验收恢复逻辑，会向 localStorage 写入模拟 session。写入后刷新页面，即可验证恢复弹窗和恢复后的状态。

当前预置场景：

- 专注中，剩余 8 秒：恢复后继续 `burning` 倒计时。
- 专注已超时，还有下一炷香：恢复后直接显示烧完提醒。
- 休息中，剩余 4 秒：恢复后继续 `resting` 倒计时。
- 休息已超时：恢复后直接显示休息结束提醒。
- 最后一炷已超时：恢复后进入 `review`。
- 旧版 session，无计时段：恢复后按保存的剩余秒数重新建立计时段。

这些开发场景只在 `import.meta.env.DEV` 下显示；正式构建不显示入口。
