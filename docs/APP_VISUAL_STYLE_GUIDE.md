# App 页面视觉风格指南

本文记录《急急如律令》当前页面级视觉基准。后续调整 UI、视觉素材、Tauri 外壳展示或自用版本前端样式时，必须先对照本文，避免页面风格偏离应用图标已经确认的方向。

## 核心方向

页面风格来自当前应用图标：米纸底、淡墨山形、青灰线条、月形留白和亮金符箓识别点。整体气质是清幽、朴素、安静、有仪式感；不走大红背景、高光法器、闪烁火焰、厚重拟物或庙宇装饰路线。

页面不是把图标元素直接复制到界面里，而是抽取图标的视觉语言：

- 米纸暖白负责承载内容。
- 淡墨青灰负责结构、边界、说明文字和次级操作。
- 亮金负责主操作、选中态、聚焦态和少量关键标签。
- 深墨色负责正文与标题，保证长期书写和复盘时可读。
- 火焰橙红只用于真实燃烧、错误或危险语义，不能再成为普通页面的主品牌色。

## 页面色彩

当前 CSS 令牌集中维护在 `src/styles.css` 的 `:root` 中。后续改色时优先改这些令牌，不在组件里新增零散品牌色。

- 主文字：`--brand-ink`，接近墨黑，不使用纯黑造成硬边。
- 次级文字：`--brand-muted`，偏青灰，服务说明、摘要和弱状态。
- 页面底色：`--brand-paper` / `--brand-paper-soft`，模拟米纸暖白。
- 面板底色：`--brand-panel-solid` / `--brand-panel-raised`，保持纸面层级，不使用纯白大面积平板感。
- 结构线：`--brand-border` / `--brand-border-strong`，低饱和、细线、弱存在感。
- 次级操作：`--brand-sage` / `--brand-sage-deep`，沿用图标青灰线条。
- 主识别色：`--brand-gold` / `--brand-gold-strong` / `--brand-gold-deep`，对应图标亮金符箓线稿。
- 错误和危险：`--brand-danger` / `--brand-danger-soft`，只用于校验、删除、放弃本轮等明确语义。

## 页面结构

- 页面背景允许有极淡纸面层次或山形斜线，但不能出现大面积圆团、渐变球、光斑、散景或与内容无关的装饰。
- 内容面板继续使用低调纸色、细边框和轻阴影；避免让页面变成一组营销卡片。
- 圆角维持克制，默认不超过 `8px`；只有 pill 标签、关闭按钮等自然需要圆形的控件使用大圆角。
- 表单、历史、设置、复盘和弹窗共享同一套面板、边框、按钮和焦点态，不为单个页面临时发明一套视觉系统。
- 页面第一视觉路径始终服务当前任务：填写页是“开始创造”，仪式台是符箓/香炉/计时，复盘页是完成度、一句复盘和保存。

## 交互层级

- 主按钮、选中 segmented control、当前打开的顶部入口使用亮金；亮金应该“一眼可见”，但数量要少。
- 次级按钮使用米纸底、青灰文字和细边框；hover 时可以短暂带亮金纸感。
- 危险按钮使用克制的红褐色，不复用亮金，也不能抢过当前主行动。
- 输入框 focus 使用亮金描边和柔和外圈，呼应符箓线条；错误态再切换到危险色。
- toast、badge、历史指标和摘要 chip 使用同一套纸色、青灰和少量亮金，不使用旧朱红作为普通强调色。

## 视觉素材关系

应用图标是页面风格源头，但图标、菜单栏图标、通知图标和仪式台素材仍然分开设计、分开验收：

- 应用图标视觉基准见 `src-tauri/icons/README.md` 和 `src-tauri/icons/app-icon/README.md`。
- 仪式台素材接入规则见 `docs/ASSET_PIPELINE.md`、`docs/VISUAL_ASSET_BOUNDARIES.md` 和 `docs/VISUAL_ASSET_REPLACEMENT_CHECKLIST.md`。
- 菜单栏 template icon 继续保持透明单色，不复用彩色应用图标或页面亮金色。

仪式台素材可以保留燃烧、火星和香炉本体所需的暖色，但这些暖色只服务场景动作，不应外溢为全页面品牌色。

## 调整前检查

后续任何页面视觉调整至少检查以下事项：

- 是否仍能从页面上看出米纸、青灰、亮金这三个主线索。
- 是否把亮金保留给主操作、选中态和关键识别点。
- 是否重新引入了大红背景、闪光香炉、模糊圆团、强装饰背景或厚重拟物。
- 桌面 `960px × 760px` 和移动约 `390px` 宽度下，文字、按钮和输入框是否无重叠、无横向滚动。
- `npm run build` 和 `git diff --check` 是否通过。
- 准备替换自用 `.app` 时，继续按 `docs/SELF_USE_REGRESSION_RUNBOOK.md` 和 `docs/MACOS_SELF_USE_INSTALL.md` 验收。
