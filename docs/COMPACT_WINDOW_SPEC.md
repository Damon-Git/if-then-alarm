# 小窗规格

当前小窗目标是为未来 macOS 菜单栏弹窗准备窄窗口体验。现阶段仍在 Web 中验收，不初始化 Tauri，不接系统菜单栏。

## 目标尺寸

- 第一验收尺寸：`390px × 620px`。
- 宽度范围：优先覆盖 `390px`，兼容 `390-420px`。
- 内容可以纵向滚动，但不应出现横向滚动。
- 弹窗内容过长时，弹窗内部滚动，不挤出视口。

## 小窗显示内容

仪式台小窗只显示用户实际创建的香炉槽位：

- 用户创建 1 套，显示 1 个香炉。
- 用户创建 2 套，显示 2 个香炉。
- 用户创建 3 套，显示 3 个香炉。
- 不补空槽位，最多 3 个。

每个香炉槽位显示：

- Q 版香炉占位，未来替换为 `compact` 素材。
- 当前状态：未开始 / 烧香中 / 休息中 / 已完成。
- 当前香数或倒计时。
- 一行短提示。
- 情境性执行意图摘要，最多作为辅助信息，不显示符箓。

## 小窗不显示内容

小窗仪式台不显示：

- 完整情境性符箓卡片。
- 预防性符箓列表。
- 主祭台三栏布局。
- 大面积视觉装饰。
- 复杂动画或烟雾。

历史和设置仍可通过右上角弱入口打开，但不应挤占仪式台第一屏。

## 交互原则

- 未开始香炉可以点击，进入同一套开始确认流程。
- 烧香中和休息中的香炉不可点击开始其他套组。
- 已完成香炉不可再次开始。
- 点击香炉不替代主祭台上的符箓燃烧逻辑；小窗只是窄窗口下的交互入口。

## 弹窗约束

小窗下所有弹窗必须满足：

- 宽度不超过视口。
- 高度不超过视口。
- 内容过长时可滚动。
- 操作按钮纵向排列，避免拥挤。
- 长文本必须换行，不遮挡按钮。

适用弹窗：

- 开始确认。
- 单炷结束后的休息/续香。
- 休息结束后的续香。
- 放弃本轮确认。
- 历史删除/清空确认。
- 恢复本轮。

## 历史与设置面板

小窗下历史和设置面板应满足：

- 搜索、筛选、导入、导出、清空按钮纵向或紧凑排列。
- 历史摘要允许换行，不横向溢出。
- 开发恢复场景按钮纵向排列。
- 面板可随页面滚动，不固定占满窗口。

## 当前不做

- 不做真实 macOS 菜单栏弹窗。
- 不初始化 Tauri。
- 不隐藏 Dock。
- 不做置顶窗口。
- 不接系统通知。
- 不接真实 Q 版香炉素材。
- 不做动画。
