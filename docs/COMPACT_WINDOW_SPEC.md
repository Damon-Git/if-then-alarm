# 小窗规格

交互语义总览见 `docs/INTERACTION_MODEL.md`。本文只细化小窗视觉和行为约束。

当前小窗目标是为未来 macOS 菜单栏弹窗准备低干扰计时器体验。小窗 ritual 阶段不是完整应用的响应式压缩版，而是只显示用户创建的 1-3 个 Q 版小香炉的香炉场景。香炉始终单行并排，不绘制额外灰色或米色背景。现阶段已经提供 Tauri 外壳、菜单栏入口和 `390px × 620px` 小窗尺寸基线；用户显式选择“保留并收起”时才切换为透明、无原生标题栏的窗口外壳。macOS 透明窗口依赖 Tauri `macOSPrivateApi`、`macos-private-api` Cargo feature 和透明窗口背景配置；当前仅作为内部工具验证，不作为 Mac App Store 发布形态。Web 窄窗口只验证完整窗口的响应式布局，不自动进入小窗模式；自动检查脚本会显式设置小窗模式来验证小窗场景。

Tauri 桌面窗口的完整手动验收路径见 `docs/TAURI_DESKTOP_CHECKLIST.md`。桌面行为组合回归顺序见 `docs/DESKTOP_BEHAVIOR_REGRESSION.md`。

## 窗口模式

当前桌面端有两个尺寸模式：

- 小窗模式：`390px × 620px`，透明、无原生标题栏，只承载小窗香炉场景。
- 完整窗口模式：`960px × 760px`，恢复原生标题栏和窗口阴影，承载主祭台、符箓、历史、设置、复盘等完整操作。

窗口行为语义：

- “保留并收起”表示从完整窗口切回小窗模式，窗口仍保持可见，计时继续。
- “进入仪式台”表示进入完整窗口模式，先展示主祭台和符箓。
- 用户手动把完整窗口拖窄时，仍停留在完整窗口模式，不自动切换为小窗香炉舞台。
- 点击小窗香炉表示展开完整窗口模式，不在小窗中直接开始或续香。
- 小窗香炉横排上方提供使用 Tauri 窗口位置会话的独立透明拖拽热区；拖拽热区不覆盖香炉按钮，拖动窗口不触发展开。
- 菜单栏 template icon 入口只负责隐藏或重新显示当前窗口，不改变窗口尺寸模式。

## 目标尺寸

- 第一验收尺寸：`390px × 620px`。
- 宽度范围：优先覆盖 `390px`，兼容 `390-420px`。
- 内容可以纵向滚动，但不应出现横向滚动。
- 弹窗内容过长时，弹窗内部滚动，不挤出视口。

## 自动验收

本地开发服务启动后，可以运行：

```bash
npm run check:compact
```

脚本会使用 `390px × 620px` 视口打开 `http://127.0.0.1:5173/`，自动创建 3 套执行意图并进入仪式台。脚本会先确认窄视口下仍显示完整仪式台，再显式切换到小窗模式，然后检查：

- 页面没有横向溢出。
- 手动窄视口不会自动进入小窗模式。
- 显式小窗模式下，小窗香炉舞台可见。
- 小窗 ritual 场景隐藏应用标题、历史按钮、设置按钮和仪式台标题。
- 小窗香炉始终单行并排。
- 小窗 ritual 场景不绘制额外背景。
- 小窗 ritual 场景的根元素、应用主壳、面板和香炉按钮都不绘制背景。
- 小窗香炉使用 `censer/compact` 图片图层，不退回纯 CSS 占位图。
- 小窗香炉 `body/lid/mouth/ash/feet` 五个图层都存在并能叠放。
- 小窗提供单一独立拖拽热区，尺寸足够抓取，并且不与香炉按钮重叠。
- 在浏览器调试环境中按下拖拽热区不会启动、续香、进入复盘或修改业务状态。
- 从香炉按钮自身按下后移动超过阈值，会抑制随后一次点击，不尝试展开完整窗口。
- 启动一套后，小窗中只有当前烧香的香炉进入更强的进行中视觉状态，其他香炉保持未开始弱状态。
- 小窗中最后一炷香结束后停留在完成香炉场景，不直接进入复盘。
- 非 Tauri 调试环境中，点击完成态香炉如果不能展开完整窗口，会继续停留在小窗完成态。
- 小窗香炉不直接显示状态文案、倒计时、提示或长执行意图。
- 完整主祭台只在显式小窗模式下隐藏。
- 香炉槽位数量与创建套组数量一致。

截图会输出到 `artifacts/compact-window.png`。`artifacts/` 不进入 Git。

浏览器自动检查可以验证拖拽热区使用窗口位置会话、布局和非业务语义，也可以验证香炉按钮在明显移动后会抑制随后一次点击。Tauri 拖拽热区是否确实移动桌面窗口、从拖拽热区拖动时是否误触香炉展开，仍必须通过 `npm run tauri:dev` 人工验收。

## 小窗显示内容

仪式台小窗只显示用户实际创建的香炉槽位：

- 用户创建 1 套，显示 1 个香炉。
- 用户创建 2 套，显示 2 个香炉。
- 用户创建 3 套，显示 3 个香炉。
- 无论几个香炉，都单行并排显示。
- 不补空槽位，最多 3 个。

每个香炉槽位保留：

- Q 版香炉 v1 `compact` 素材。
- 必要的按钮可访问性文案。
- 当前状态、倒计时和意图摘要只作为辅助信息保留在交互语义中，不在小窗香炉场景里直接显示。

## 小窗状态表达

小窗状态表达只服务于“低干扰计时器”：

- 未开始：香炉保持普通弱状态。
- 烧香中：当前香炉比其他香炉更明显，用于标识正在进行的套组。
- 休息中：香炉和线香弱化，避免看起来仍在积极燃烧。
- 已完成：香炉保持稳定弱化的完成态。

状态表达只使用图层透明度、饱和度、对比度和非布局缩放，不新增文字、不新增背景面板、不做动画。

## 小窗不显示内容

小窗仪式台不显示：

- 应用大标题。
- 历史和设置入口。
- 仪式台标题。
- 放弃本轮按钮。
- 完整情境性符箓卡片。
- 预防性符箓列表。
- 主祭台三栏布局。
- 情境性执行意图摘要。
- 说明性状态文案。
- 大面积视觉装饰。
- 灰色或米色背景面板。
- 复杂动画或烟雾。

历史、设置、完整主祭台、符箓和复盘等完整操作只属于完整状态，不属于小窗香炉场景。

## 交互原则

- 未开始香炉可以点击，但只用于展开完整窗口，不打开开始确认。
- 烧香中和休息中的香炉点击后尝试展开并聚焦完整窗口，方便处理休息、续香、复盘等完整操作。
- 已完成香炉点击后尝试展开并聚焦完整窗口，方便查看完整状态。
- 拖动香炉横排上方的独立热区只移动小窗，不展开完整窗口，也不改变任何业务状态。
- 香炉按钮自身仍支持正常单击展开；如果从按钮按下后移动超过阈值，则抑制随后一次点击，避免拖动手势误展开。拖拽热区不覆盖香炉主要点击入口。
- 全部香完成时，小窗不直接弹出复盘页，也不显示“本轮香尽”或“进入复盘”收尾入口，只保留已完成香炉场景；香炉按钮语义提示“点击展开完整窗口查看复盘入口”。展开完整窗口后仍停留完成祭台场景，由用户在完整窗口点击“进入复盘”。
- 全部香完成且当前处于小窗时，系统通知文案应指向“点击香炉回到完整窗口复盘”，避免误导用户在小窗内处理复盘。
- 如果已有其他套组正在烧香或休息，点击未开始香炉也应展开完整窗口，而不是启动新的套组。
- 点击香炉不替代主祭台上的符箓燃烧逻辑；小窗只是低干扰计时器和返回完整窗口的入口。
- Web 窄窗口调试环境没有真实桌面窗口可展开；该动作在非 Tauri 环境中安全跳过。

## 弹窗约束

小窗下所有弹窗必须满足：

- 宽度不超过视口。
- 高度不超过视口。
- 内容过长时可滚动。
- 操作按钮纵向排列，避免拥挤。
- 长文本必须换行，不遮挡按钮。

适用弹窗：

- 单炷结束后的休息/续香。
- 休息结束后的续香。
- 放弃本轮确认。
- 历史删除/清空确认。
- 恢复本轮。

## 历史与设置面板

小窗 ritual 阶段不显示历史和设置入口。小窗非 ritual 阶段如果打开历史或设置面板，应满足：

- 搜索、筛选、导入、导出、清空按钮纵向或紧凑排列。
- 历史摘要允许换行，不横向溢出。
- 开发恢复场景按钮纵向排列。
- 面板可随页面滚动，不固定占满窗口。

## 当前不做

- 不做真正贴附菜单栏图标的 popover 窗口；当前是由菜单栏入口显示或隐藏普通 Tauri 小窗。
- 不默认隐藏 Dock；Tauri 桌面版提供用户可选的“在 Dock 中显示”设置。
- 不默认置顶窗口；Tauri 桌面版提供用户可选的“窗口置顶”设置。
- 不在当前阶段继续制作第二轮 Q 版香炉素材。
- 不做动画。

## 2026-06-02 Tauri 实测记录

- 烧香中选择“保留并收起”后，原生窗口切换为 `390px × 620px` 小窗，仍保留当前香炉的 active 视觉状态。
- 从独立拖拽热区拖动后，窗口从 `(540, 109)` 移到 `(640, 189)`，尺寸保持 `390px × 620px`，没有误触展开。
- 正常点击小窗香炉后，窗口恢复完整模式；点击只负责展开，不启动、不续香、不进入复盘。
- 恢复完整模式后，原生窗口外框重新接近 `960px × 760px`；运行时 `setSize` 使用 client area 尺寸，适配器会扣除重新显示的原生标题栏高度，避免每次从小窗展开时额外增高。
- 修复后已连续两次完成 `390px × 620px → 960px × 760px` 原生窗口往返，外框尺寸稳定；本次修改结果已由用户验证通过。
- 小窗继续保持低干扰：不启用主祭台新增烟缕动画，不显示计时卡、香炉信息卡、符箓、复盘入口或背景面板。
