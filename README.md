# 急急如律令

《急急如律令》是一款优先适配 macOS 的独立桌面小窗工具。项目采用自用优先的开源路线；当前保留 Web App 开发入口，并已接入 Tauri 桌面外壳用于内部 `.app` 冒烟。

产品使用“符箓 + 香 + 香炉”的中国文化视觉隐喻承载执行意图和番茄钟流程。它不宣称宗教、玄学或超自然功效，核心目标是用仪式感帮助用户启动行动。

## 当前范围

当前版本是内部自用 MVP，优先跑通完整业务流程，并已完成仪式台视觉素材 v1 收尾：

- 创建 1-3 项任务。
- 每项任务包含 1 条目标性执行意图、0-3 条预防性执行意图、1-3 炷香。
- 目标性执行意图使用“当……就……”轻校验，并提供书写提示和字段级错误定位。
- 预防性执行意图使用“如果……那么……”轻校验，并提供阻碍与替代动作提示。
- 仪式台中点击目标性符箓会先确认本项任务内容，再开始倒计时。
- 单炷香结束后选择休息或续香。
- 全部完成后填写复盘，复盘页可回看本轮详情，并记录完成度、主要阻碍和下次调整。
- Web 环境下复盘写入 localStorage；Tauri 桌面环境下写入 app data 目录下的桌面 JSON。
- 历史记录以折叠摘要展示复盘字段，支持删除单条和清空全部。
- 历史记录顶部展示轻量分析：总轮次、完成度、总香数、累计专注、阻碍 Top 5 和最近下次调整。
- 历史记录支持搜索、完成度筛选和时间范围筛选，分析面板会跟随筛选结果更新。
- 历史记录支持导出 JSON 备份，也支持导入本应用导出的 JSON 文件并按 id 去重追加。
- 复盘保存、历史导入导出、删除和清空使用轻量 toast 提示。
- 支持开发模式和正式模式切换，设置会随当前运行环境保存到 Web localStorage 或 Tauri 桌面 JSON。
- 设置中提供声音提醒开关，开启后使用 Web Audio 播放临时钟声。
- 历史和设置收为右上角弱入口，默认不挤占主流程。
- 点击“开始创造”后进入完整窗口和主祭台；用户手动拖窄完整窗口时不自动切换为小窗。
- 显式选择“保留并收起”后进入小窗香炉舞台：只显示用户创建的 1-3 个香炉槽位，点击香炉只用于展开完整窗口，不直接开始或续香。
- 小窗下隐藏标题和辅助文案，只保留低干扰香炉计时场景。
- 主祭台和小窗香炉统一通过 `CenserVisual` 渲染，分别使用 `stage` 和 `compact` 正式 v1 素材族。
- 目标性和预防性符箓统一通过 `TalismanVisual` 渲染，并已接入两套正式 v1 图片模板。
- 线香统一通过 `IncenseVisual` 渲染，分别使用主祭台和小窗正式 v1 图层，并接收由倒计时计算出的当前炷香进度。
- 仪式台和复盘页提供放弃本轮、刷新/关闭保护。
- 填写页已有草稿时，刷新或关闭页面会触发浏览器保护提示。
- 未完成轮次会暂存到 localStorage，刷新后可选择恢复或丢弃。
- 历史记录当前只用于回看、分析和反思，不提供一键复用。

当前不继续扩展复杂视觉动画，不接后端，不做 DMG、签名、公证或 Mac App Store 上架，也不购买 Apple Developer Program 会员。公开分享优先提供源码、构建说明和数据安全说明；仪式台七组视觉目标已完成 v1，后续视觉工作属于新阶段迭代。

开源许可证尚未确定。在添加明确许可证并完成素材授权与敏感信息审计前，本仓库不应对外描述为已经完成授权的开源发布。

## 运行

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

构建验证：

```bash
npm run build
```

内部 macOS `.app` 冒烟：

```bash
npm run tauri:build
```

内部测试包会生成在：

```text
src-tauri/target/release/bundle/macos/急急如律令.app
```

## 开发计时常量

当前默认使用开发模式，方便测试：

```ts
DEV_TIMER_SECONDS = 10
DEV_BREAK_SECONDS = 5
```

未来正式版可切换为：

```ts
PROD_TIMER_SECONDS = 25 * 60
PROD_BREAK_SECONDS = 5 * 60
```

常量定义在 `src/constants.ts`。

计时模式：

- 开发模式：10 秒专注 / 5 秒休息。
- 正式模式：25 分钟专注 / 5 分钟休息。

计时模式会保存到 localStorage。点击“开始创造”进入仪式台后、第一张符箓开始燃烧前，仍允许从右上角设置切换模式；一旦确认开始并进入符箓燃烧动画，或存在待恢复轮次，就不允许切换模式。完成、放弃或丢弃当前轮次后可以切换。session 暂存和历史记录会保存本轮使用的计时模式。

## 数据备份

历史记录保存在浏览器 localStorage 中。可以在历史记录面板导出 JSON 备份，文件名格式为：

```text
jiji-rululing-history-YYYYMMDD.json
```

导入时只接受本应用导出的 JSON 格式。导入策略为追加导入，并按历史记录 `id` 跳过重复记录。

## 产品原则

当前阶段刻意保留重新书写执行意图的动作。历史记录不提供“再次使用”入口，避免把行动承诺变成无感复制；后续如果进入固定习惯场景，再评估是否增加复用能力。

## 协作约定

- 文档使用中文，文件名使用英文。
- 代码文件名使用英文。
- 不自动提交 Git。
- 敏感信息放到 `.env.local`，不要进入 Git。
- 当前阶段以流程完整和状态可靠为第一优先级。
