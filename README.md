# 急急如律令

《急急如律令》是一款优先适配 macOS 的独立桌面小窗工具。当前阶段先以 Web App 方式实现 MVP，后续再迁移到 Tauri。

产品使用“符箓 + 香 + 香炉”的中国文化视觉隐喻承载执行意图和番茄钟流程。它不宣称宗教、玄学或超自然功效，核心目标是用仪式感帮助用户启动行动。

## 当前范围

当前版本是无动画 MVP，优先跑通完整业务流程：

- 填写 1-3 套执行意图。
- 每套包含 1 条情境性执行意图、0-3 条预防性执行意图、1-3 炷香。
- 情境性执行意图使用“当……就……”轻校验，并提供书写提示和字段级错误定位。
- 预防性执行意图使用“如果……那么……”轻校验，并提供阻碍与替代动作提示。
- 进入仪式台后点击情境性符箓会先确认本套内容，再开始倒计时。
- 单炷香结束后选择休息或续香。
- 全部完成后填写复盘，复盘页可回看本轮详情，并记录完成度、主要阻碍和下次调整。
- 复盘写入 localStorage 历史记录。
- 历史记录以折叠摘要展示复盘字段，支持删除单条和清空全部。
- 历史记录顶部展示轻量分析：总轮次、完成度、总香数、累计专注、阻碍 Top 5 和最近下次调整。
- 历史记录支持搜索、完成度筛选和时间范围筛选，分析面板会跟随筛选结果更新。
- 历史记录支持导出 JSON 备份，也支持导入本应用导出的 JSON 文件并按 id 去重追加。
- 复盘保存、历史导入导出、删除和清空使用轻量 toast 提示。
- 支持开发模式和正式模式切换，设置会保存到 localStorage。
- 仪式台和复盘页提供放弃本轮、刷新/关闭保护。
- 填写页已有草稿时，刷新或关闭页面会触发浏览器保护提示。
- 未完成轮次会暂存到 localStorage，刷新后可选择恢复或丢弃。
- 历史记录当前只用于回看、分析和反思，不提供一键复用。

当前不做复杂视觉、动画、音效、真实素材替换、后端和桌面打包。

## 运行

安装依赖：

```bash
npm install
```

启动开发服务：

```bash
npm run dev
```

构建验证：

```bash
npm run build
```

## 开发计时常量

当前默认使用开发模式，方便测试：

```ts
DEV_TIMER_SECONDS = 10
DEV_BREAK_SECONDS = 5
```

未来正式版可切换为：

```ts
PROD_TIMER_SECONDS = 25 * 60
PROD_BREAK_SECONDS = 5 * 60
```

常量定义在 `src/constants.ts`。

计时模式：

- 开发模式：10 秒专注 / 5 秒休息。
- 正式模式：25 分钟专注 / 5 分钟休息。

计时模式会保存到 localStorage。当前轮次进行中或有待恢复轮次时，不允许切换模式；完成、放弃或丢弃当前轮次后可以切换。session 暂存和历史记录会保存本轮使用的计时模式。

## 数据备份

历史记录保存在浏览器 localStorage 中。可以在历史记录面板导出 JSON 备份，文件名格式为：

```text
jiji-rululing-history-YYYYMMDD.json
```

导入时只接受本应用导出的 JSON 格式。导入策略为追加导入，并按历史记录 `id` 跳过重复记录。

## 产品原则

当前阶段刻意保留重新书写执行意图的动作。历史记录不提供“再次使用”入口，避免把行动承诺变成无感复制；后续如果进入固定习惯场景，再评估是否增加复用能力。
