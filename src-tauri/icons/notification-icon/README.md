# 通知图标

本目录用于未来系统通知图标策略和素材。

## 用途

- macOS 系统通知中的应用识别。
- 未来如需区分通知样式，可在此目录记录通知图标资产。

## 当前状态

当前没有单独通知图标素材。`src/lib/notificationAdapter.ts` 发送通知时只传入 `id`、`title` 和 `body`，系统通知使用 Tauri/macOS 默认应用图标策略；应用图标 v1 接入后需要单独验收通知中的应用识别表现。

只有 macOS 实测证明需要额外素材时，再为本目录定义通知图标文件和接入方式。

2026-06-01 自用验收中，通知正文和触发时机正常，但通知横幅仍可能显示旧红色圆形占位图。已排查 LaunchServices 旧路径索引并重启当前用户的通知中心 UI agent，问题仍复现。由于不影响个人自用核心流程，该问题当前为最低优先级，暂缓继续处理。

## 未来验收标准

- 通知中小尺寸可读。
- 不依赖细小文字。
- 与应用图标关系明确：可以沿用应用图标，但必须单独验收。
- 不复用菜单栏 template icon。

## 当前不做

- 不制作通知图标。
- 不为不同通知类型设计多套图标。
- 不把占位应用图标视为正式通知图标。
